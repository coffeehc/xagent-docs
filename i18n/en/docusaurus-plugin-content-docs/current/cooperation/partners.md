# Ecosystem Partners

xAgent is currently in `v0.0.1.beta`. The free binary release is intended to let more users deploy, try, and provide feedback from real task scenarios.

As xAgent is used for long-running tasks, file processing, tool calling, and automated collaboration, model cost and external capability ecosystems become important. We are looking for partners who can reduce long-task running cost and expand real-world scenario coverage.

## Model Provider Partnership

xAgent is looking for a model API or token provider as a recommended default model-service partner.

An ideal partner can provide:

- Stable model API service.
- High-quality mainstream open-source models.
- Stable OpenAI-compatible API or a clear integration protocol.
- Long context, concurrent calls, and long-task support.
- Clear quota, rate-limit, and billing rules.
- Model combinations suitable for long tasks, tool calling, file understanding, and report generation.
- Exclusive discounts, trial quota, or tiered pricing for xAgent users.

xAgent does not want to lock users into one provider. Different models have different strengths, and future xAgent model routing will continue to improve based on testing.

However, long-running tasks, repeated file organization, tool calls, and deliverable generation all create ongoing model cost. A stable, easy-to-use, cost-effective default model entry can significantly reduce the pressure of getting started and using xAgent long term.

## MCP Ecosystem Partnership

xAgent is also looking for high-quality MCP aggregation providers to improve scenario coverage.

An ideal partner can provide:

- A rich and maintainable MCP Server directory covering office, development, data, operations, and other frequent scenarios.
- Unified authorization, stable connection, service health state, and call observability.
- Clear invocation, billing, and security boundaries.
- Ability to combine with xAgent Skill system and turn connection capabilities into user-facing scenario templates.
- Continuous supply of high-quality MCP services.

In xAgent, Skills describe task methods and execution flows. MCP provides external callable capabilities. Good MCP aggregation can work with Skills: the Skill says how a task should be done, and MCP provides what can be called during execution.
