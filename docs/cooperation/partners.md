# 生态合作

xAgent 当前处于 `v0.0.4.beta` 测试版阶段，免费二进制版本的目标是让更多用户先部署、体验并反馈真实任务场景。随着用户开始把 xAgent 用于长任务、文件处理、工具调用和自动化协作，模型调用成本和外部能力生态会变得越来越重要。

因此，我们希望寻找能够降低长期任务运行成本、扩展场景生态的合作伙伴，一起完善 xAgent 的默认使用体验和实际落地能力。

## 模型供应商合作

xAgent 希望寻找一家模型 API 或 Token 供应商，作为默认推荐的模型服务合作方。

理想的合作方可以提供：

- 稳定可用的模型 API 服务。
- 覆盖主流高质量开源模型，提供稳定的 OpenAI-compatible API 或清晰的接入协议。
- 支持长上下文、并发调用和持续任务运行，具备清晰的配额、限流和账单规则。
- 适合长任务、工具调用、文件理解、报告生成等工作场景的模型组合。
- 面向 xAgent 用户提供专属折扣、试用额度或阶梯价格，降低长期任务运行成本。

xAgent 的目标不是绑定用户只能使用某一家模型服务。不同模型各有所长，后续 xAgent 也会持续优化模型路由，让不同任务使用更适合的模型。

但从实际使用角度看，长期运行任务、持续整理材料、反复调用工具和生成交付产物都会产生持续的模型调用成本。如果能为用户提供一个稳定、易用且更具性价比的默认模型入口，会显著降低用户开始使用和长期使用 xAgent 的压力。

## MCP 生态合作

xAgent 也希望寻找优秀的 MCP 聚合服务商，共同完善用户的场景生态。

理想的合作方可以提供：

- 丰富、可维护的 MCP Server 目录，覆盖办公、研发、数据、运营等高频场景。
- 统一授权、稳定连接、服务健康状态和调用可观测能力，适合长期任务链路使用。
- 清晰的调用、计费和安全边界。
- 能够与 xAgent Skill 体系结合，把连接能力进一步沉淀为面向用户的场景模板。
- 能够持续补充高质量 MCP 服务，帮助用户更快搭建可用的专用智能体场景。

在 xAgent 中，Skill 更偏向任务方法和执行流程，MCP 更偏向外部可调用能力。优秀的 MCP 聚合服务可以和 Skill 形成配合：Skill 描述“某类任务应该怎么做”，MCP 提供“任务执行时可以调用哪些外部能力”。

这类合作可以帮助用户更快完成场景落地，例如资料检索、文档处理、业务系统查询、消息发送、项目协作、内容生成和多工具联动等。

## 生态合作方式

当前阶段，我们更关注真实可用、边界清晰、能帮助用户降低成本或提高场景覆盖度的合作。欢迎开放沟通，不要求一开始就形成固定商业方案。

欢迎以下类型的合作方联系：

- 模型 API / Token 服务商。
- 开源模型 API 聚合服务商。
- MCP 聚合服务商。
- 能提供稳定外部能力、并愿意配合 xAgent Skill 生态建设的服务方。

如果你是模型 API / Token 服务商，或正在建设 MCP 聚合服务，欢迎联系 xAgent 团队。如果你希望讨论合作，请通过邮件联系：

**data@yaoankeji.com**

来信时建议简单说明：

- 公司或产品名称。
- 可提供的模型、Token、MCP 目录或外部能力范围。
- 计费方式、服务稳定性、价格或优惠方案。
- 是否已有公开文档、API 文档或测试入口。
- 你希望与 xAgent 形成的合作方式。
