# Error Codes

> 状态：规划中。

## 适用对象

本文适合需要排查 xAgent 错误提示的用户。

## 这是什么

Error Codes 用于集中记录稳定错误码、错误含义、触发条件和处理建议。当前稳定错误码清单尚未整理完成，本页先记录排查口径。

## 什么时候使用

当会话、登录、工具调用、文件处理或外部连接失败时，先按本文方向排查。

## 基本用法

排查顺序：

1. 看页面或会话中的原始错误提示。
2. 判断失败发生在登录、模型、Tool、Workspace、Connector、Trigger 还是配置。
3. 检查当前用户是否有权限、配置是否完整、目标资源是否存在。
4. 如果是 Tool 失败，查看工具结果中的状态、参数提示和可恢复建议。
5. 如果错误稳定复现，再记录错误码、页面、输入、时间和相关 Session。

后续补充稳定错误码时，每个条目应包含错误码、所属模块、含义、触发条件和用户可执行处理方式。

## 常见问题

### 可以先写常见错误文案吗？

可以写排查方向，但不能编造错误码。

### 错误码是否需要稳定格式？

需要。稳定后应在本文集中维护。

### 错误码变化是否需要 Changelog？

需要。
