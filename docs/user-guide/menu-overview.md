# 功能导览与菜单入口

## 适用对象

本文适合第一次进入 xAgent 控制台，希望先知道“有哪些功能、从哪里进入、能完成什么”的用户。

## 这是什么

本文把功能导览和菜单说明合并在一起。先用完整界面截图建立整体印象，再按实际使用路径说明 xAgent 能做什么，最后对照左侧菜单说明每个入口的用途。

xAgent 的使用主线是：管理员先准备常用入口和可用能力，普通用户直接进入网页提交任务、补充材料、确认动作并查看结果。

## 什么时候使用

第一次打开 xAgent、给新用户介绍功能、或不知道某件事该去哪个页面完成时，先阅读本文。

## 使用路径

不同角色可以按下面路径开始：

- 普通用户：进入 Agent会话，说明要完成什么、材料在哪里、希望得到什么结果；也可以直接使用 Skill、Tool、连接器、触发器和智能体配置，只是不需要从零理解和配置它们。熟悉后，可以在个人范围内创建或精调 Skill、智能体、MCP、密钥和审批规则。
- 管理员：先准备账号、模型、常用工作入口、工具、技能、外部连接和安全策略。

## 常用菜单入口

如果你不确定该打开哪个页面，可以先按下面的菜单名查找。这里的“入口”都对应左侧菜单里的真实名称。

| 想做什么 | 打开哪个菜单 | 适合场景 |
| --- | --- | --- |
| 提交任务、上传材料、查看执行过程 | [Agent会话](/docs/user-guide/agent-session) | 阅读文件、整理内容、生成报告、持续追问、确认风险动作。这个页面逻辑较多，建议单独阅读说明 |
| 查看上传文件、生成文件和最终产物 | 工作区文件 | 预览 Markdown、HTML、表格和报告，下载或继续引用生成产物 |
| 查找和管理历史对话 | 会话列表 | 搜索会话、进入主会话或子会话、查看目标和状态 |
| 查看需要人工确认的动作 | 审批 | 查看审批记录、进入详情、追踪风险等级和关联会话 |
| 让任务定时或由外部事件启动 | 触发器管理 | 每天或每周生成固定报告，到点检查邮箱，外部事件到达后自动进入会话处理 |
| 选择或调整工作助手入口 | 智能体管理 | 查看个人智能体和公共智能体，创建个人入口，选择默认 Skill、Tool 和密钥 |
| 复用固定工作方法 | Skill 管理 | 使用、创建或维护研究、写作、报告、客服、合同审查等固定流程 |
| 查看可用执行能力 | 我的工具 | 查看当前用户可见工具、个人 MCP 工具、工具来源、可用状态和详情 |
| 接入个人 MCP 服务 | 我的 MCP | 新建 MCP、发现工具、查看协议、地址、工具数和状态 |
| 接入外部账号或消息渠道 | 我的连接 | 绑定微信、邮件或其他连接器，查看认证状态、通道状态和可用工具 |
| 保存个人密钥 | 密钥管理 | 保存 API Key、邮箱密码、外部系统 token 等密钥引用 |

## 工作台

工作台是普通用户最常使用的一组入口。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| 仪表板 | 查看控制台运行概况 | 查看 token 使用统计、会话状态、等待审批数量和最近会话 |
| [Agent会话](/docs/user-guide/agent-session) | 提交需求和查看执行过程 | 输入任务、上传附件、查看回复、确认工具动作、继续追问；详细逻辑见单独页面 |
| 工作区文件 | 查看当前用户可见的工作区文件和产物 | 浏览业务空间、Agent 会话目录和个人 Skill 目录，上传文件、预览文件、下载原始文件 |
| 会话列表 | 统一管理主会话和子会话 | 搜索会话、进入对话、查看状态和目标、修改标题、删除无用会话 |

建议普通用户从 Agent会话 开始。这里的 Agent 可以简单理解为“工作助手”，不需要理解技术含义。如果任务涉及文件，再进入工作区文件；如果要查历史，再进入会话列表。

当前版本的“工作编排”仍在开发中，侧边栏默认不展示，不作为本手册的主要使用入口。

## 运行治理

运行治理用于管理执行过程中的风险动作、自动触发、可用能力和外部连接。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| 审批 | 查看当前用户相关的敏感动作审批 | 查看审批记录、进入详情、追踪风险等级和关联会话 |
| [触发器管理](/docs/user-guide/trigger) | 管理用户级长期触发器 | 新增触发器、启用或禁用触发器、手动触发一次、查看下次运行时间和触发次数 |
| [智能体管理](/docs/user-guide/agent-management) | 管理当前用户可用的智能体定义 | 查看个人智能体和公共智能体，新建个人智能体，选择可用 Skill、Tool 和密钥 |
| [Skill 管理](/docs/user-guide/skill) | 管理当前用户可见的 Skill | 查看 Skill、进入详情、新增 Skill、编辑可修改 Skill、引用只读 Skill |
| [我的工具](/docs/user-guide/tool) | 查看当前用户可见工具 | 查看个人 MCP 工具、个人工具开关、工具来源、可用状态和工具详情 |
| 我的 MCP | 管理当前用户自己的 MCP 服务 | 新建 MCP、保存配置、自动发现工具、查看协议、地址、工具数和状态 |
| [我的连接](/docs/user-guide/connector) | 管理当前用户侧的连接器通道 | 创建连接、查看认证状态、通道状态和可用工具，按连接器要求完成扫码或授权 |
| 密钥管理 | 保存当前用户的 workspace secret | 新增密钥、更新密钥说明或值、删除不再使用的密钥 |

这里的页面不要求普通用户理解底层协议。只要知道：审批用于风险控制，触发器用于自动提交任务，工具和技能用于扩展能力，连接和密钥用于接入外部系统。

## 个人设置

个人设置只影响当前用户自己的使用体验和个人策略。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| 偏好设置 | 设置当前用户默认交互偏好 | 设置回复语言、回复详细程度和工作流风格 |
| 个人审批策略 | 设置当前账号的审批覆盖规则 | 维护个人规则，未命中或继承时继续使用系统审批策略 |
| 主题设置 | 调整当前浏览器里的控制台显示 | 切换跟随系统、亮色或暗色模式，选择配色和字体密度 |
| 邮箱管理 | 管理当前用户邮箱账号 | 添加邮箱、选择登录密钥、测试发送、设置默认邮箱 |

## 用户管理

用户管理只对管理员显示，用于维护账号和组织身份。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| 用户账号 | 管理本地账号、角色、用户组和账号级运行环境 | 新增用户、编辑资料、重置密码、格式化用户数据、删除账号 |
| 用户组 | 管理用于授权和数据隔离的本地用户组 | 创建用户组、编辑说明、删除不再使用的用户组 |

企业账号接入入口尚未开发完成，不作为当前版本的可见菜单说明。

## Agent 治理

Agent 治理只对管理员显示，用于维护系统级能力和接入状态。普通用户可以忽略这一组菜单。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| [智能体管理](/docs/user-guide/agent-management) | 管理系统级智能体定义 | 新建智能体、从运行会话沉淀定义、查看默认 Skill、Tool 和密钥配置 |
| 外接系统 | 查看外部能力治理事实 | 按 MCP、连接器、插件包、RAG、未就绪和风险筛选，查看能力数量、状态、治理、权限和生命周期 |
| [审批策略](/docs/user-guide/approval-policy) | 管理系统级审批规则 | 维护按顺序匹配的审批规则，设置直接放行、需要审批或拒绝 |
| [Skill 管理](/docs/user-guide/skill) | 管理系统可见 Skill 和提交审批 | 查看全部可见 Skill、处理提交审批、进入 Skill 详情、编辑有权限的 Skill |
| [Tool 管理](/docs/user-guide/tool) | 管理系统返回的全部工具治理信息 | 查看工具来源、启用状态、输入 schema、输出合同和治理约束 |
| [Connector 管理](/docs/user-guide/connector) | 管理系统级 Connector catalog | 添加 Connector、读取 Connector Card、探测 health、查看协议、登录流程、工具和心跳状态 |
| MCP 配置 | 管理全局 MCP 服务 | 新建 MCP、保存配置、自动探测协议和工具清单、触发工具重载 |
| 系统组件 | 查看和检测服务端运行环境 | 查看必需能力、就绪状态和缺失/异常项，测试或安装运行环境组件 |

## 系统配置

系统配置只对管理员显示，用于维护服务运行所需的基础配置。

| 菜单 | 用途 | 常见操作 |
| --- | --- | --- |
| [模型配置](/docs/user-guide/model-config) | 管理模型和 provider 连接信息 | 新建模型、测试连接、保存模型、调整能力开关和默认策略 |
| 系统配置 | 管理 `config.yml` 中的系统级字段 | 修改服务监听地址、系统邮件配置和基础安全配置；`root_dir` 运行中只读展示 |
| Agent 角色配置 | 管理固定角色配置 | 分别维护主执行链、子 Agent 蓝图补齐、索引构建、摘要和压缩等角色字段 |

## 常见问题

### 为什么我的菜单比截图少？

菜单会按账号权限裁剪。普通用户通常不会看到用户管理、Agent 治理和系统配置。

### 页面里看到真实邮箱、token 或内部地址怎么办？

不要把包含敏感信息的截图直接放进文档或公开渠道。截图前应切换到示例环境，或对敏感区域打码。

### 第一次使用应该看哪个菜单？

先进入 Agent会话。需要查看文件时进入工作区文件；需要自动执行时进入触发器管理；需要管理个人能力时进入 Skill 管理、我的工具、我的连接或密钥管理。

### 第一条任务应该怎么写？

第一条任务建议写清楚目标、材料、约束和交付要求。

示例：

```text
请阅读我上传的需求文档，整理一份功能清单，按优先级分组，并把结果保存为 Markdown 报告。
```
