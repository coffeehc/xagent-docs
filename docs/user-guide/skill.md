# 技能

## 适用对象

本文适合希望让 xAgent 稳定处理某类重复工作的用户和管理员。

## 这是什么

技能是可复用的工作方法。它把固定步骤、输出要求、注意事项和质量标准保存下来，让 xAgent 在处理同类任务时不必每次从头理解。

普通用户可以直接使用管理员准备好的技能，也可以在权限允许时创建或导入个人技能。区别只是大多数用户不需要配置，直接使用即可；熟悉后可以精调个人技能。管理员可以把稳定技能沉淀为公共能力。

![xAgent Skill 管理](/img/manual/xagent-skill-management.png)

## 什么时候使用

下面场景适合使用技能：

- 任务经常重复。
- 每次都有固定步骤和格式。
- 需要稳定输出质量。
- 需要组合多个工具完成工作。
- 需要把个人经验沉淀给团队复用。

常见技能包括研究整理、报告生成、会议纪要、合同审查、数据分析、客服回复、投研分析、文档理解等。

## 页面怎么看

Skill 管理 页面会展示技能卡片。常见信息包括：

| 信息 | 含义 |
| --- | --- |
| 技能名称 | 技能的显示名称或标识 |
| 描述 | 技能适合处理什么任务 |
| 可编辑 / 只读 | 当前用户是否可以修改 |
| 详情 | 查看技能说明、使用方式和配置 |
| 下载 / 发送 / 删除 | 针对可编辑技能的管理操作 |

普通用户主要看名称、描述和详情。不要只按名称选择技能，要看它是否适合当前任务。

## 基本用法

### 使用已有技能

1. 打开 Skill 管理。
2. 搜索任务关键词，例如“合同”“报告”“客服”。
3. 打开技能详情，确认用途是否匹配。
4. 回到 Agent会话，提交任务并说明目标和材料。
5. 如果需要使用某个技能，可以直接写“请按某某技能的方法处理”。

示例：

```text
请按“会议纪要”类方法整理我上传的录音转写，输出决策、待办、负责人和截止时间。
```

### 创建个人技能

适合已经熟悉固定流程、希望精调个人能力的用户。创建前先准备：

- 这个技能解决哪类任务。
- 用户需要提供哪些材料。
- 处理步骤是什么。
- 输出格式是什么。
- 哪些情况需要人工确认。
- 哪些内容不应该写入技能。

不要把临时文件路径、账号密码、token、验证码或一次性客户信息写进技能。

### 让技能更好用

一个好技能通常具备：

| 要素 | 说明 |
| --- | --- |
| 明确用途 | 一句话说明适合什么任务 |
| 输入要求 | 用户需要提供哪些材料 |
| 操作步骤 | 按顺序说明怎么处理 |
| 输出标准 | 结果格式、字段、质量要求 |
| 风险提醒 | 什么时候要确认或停止 |
| 示例 | 给普通用户一个可复制的任务写法 |

如果技能越来越复杂，建议拆成多个小技能，而不是把所有场景塞进一个技能。

## 常见场景

### 固定报告

```text
请按“数据可视化报告生成器”的方式，基于 sales.csv 生成一份本周经营分析报告。
```

### 客服回复

```text
请按客服支持流程，根据客户问题生成回复草稿，先不要发送。
```

### 合同审查

```text
请按合同审查方法阅读这份合同，重点检查付款、交付、违约和争议解决条款。
```

### 生成新技能草稿

```text
请根据我下面这套固定工作方法，帮我整理成一个可维护的 Skill 草稿。
```

## 管理建议

- 公共技能要面向多人，避免写个人习惯。
- 技能描述要给使用者看，不要只写内部术语。
- 常用技能应配清楚示例任务。
- 涉及外部发送、删除、账号授权的步骤，要写明先确认。
- 过期或不稳定技能应标记清楚，避免普通用户误用。

## 常见问题

### 技能是否等同于工具？

不是。工具执行动作，技能沉淀方法。技能可以指导 xAgent 使用工具，但不替代工具权限。

### 技能越多越好吗？

不是。技能太多会让用户难以选择。应优先沉淀高频、稳定、有明确价值的流程。

### 普通用户必须选择技能吗？

不必须。管理员可以把技能关联到智能体，普通用户直接提交任务即可。

### 技能修改后是否立即生效？

通常保存后即可在后续任务中使用。已经运行中的任务是否采用新内容，以当前页面和会话行为为准。
