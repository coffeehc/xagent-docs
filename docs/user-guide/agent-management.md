# xAgent 智能体管理：主 Agent 与子 Agent

## 适用对象

本文适合需要选择、查看或维护工作入口的用户和管理员。

## 这是什么

智能体管理用于维护不同场景下的工作入口。普通用户可以把它理解为“不同类型的工作助手列表”：有的适合通用问答，有的适合生成 Skill，有的适合处理某类固定业务。

![xAgent 智能体管理页面，显示可用智能体、能力数量和创建入口](/img/manual/xagent-agent-management.png)

## 什么时候使用

下面情况适合打开智能体管理：

- 想查看当前有哪些可用工作入口。
- 想选择更适合某类任务的助手。
- 管理员需要新增、调整或公开一个常用入口。
- 想确认某个入口是否带有固定技能、工具或密钥引用。

普通用户通常不需要频繁修改智能体。管理员准备好公共智能体后，用户直接在会话里选择或使用即可。

## 页面怎么看

页面中的每张卡片代表一个可用入口。常见信息包括：

| 信息 | 含义 |
| --- | --- |
| 名称 | 用户在页面中看到的入口名称 |
| 说明 | 这个入口适合处理什么任务 |
| 公共 / 个人 | 公共入口面向多人，个人入口只影响当前用户 |
| 技能数量 | 入口默认关联的任务方法数量 |
| 工具数量 | 入口默认可使用的动作能力数量 |
| 密钥数量 | 入口引用的安全凭据数量 |
| 只读 | 当前用户只能查看或使用，不能直接修改 |

## 基本用法

### 查看可用入口

1. 打开 智能体管理。
2. 使用搜索框输入名称、提示词、技能、工具或密钥关键字。
3. 通过“个人 / 公共”数量判断当前列表范围。
4. 点击卡片查看详情。

如果你只是普通用户，重点看名称和说明，选择最贴近任务的入口即可。

### 选择入口处理任务

进入 Agent会话 后，选择合适的智能体再提交任务。选择时可以按下面规则判断：

| 任务类型 | 建议选择 |
| --- | --- |
| 不确定归类 | 通用智能体 |
| 需要创建或优化 Skill | Skill 生成专家 |
| 需要固定业务流程 | 管理员准备的业务专用智能体 |
| 需要外部系统能力 | 已绑定相关工具或连接的智能体 |

如果不确定该选哪个，优先选择通用入口，并在任务里说明目标。

### 管理员新建入口

管理员创建智能体时，建议先准备这些内容：

- 名称：让用户一眼知道用途。
- 简短说明：说明适合处理哪些任务。
- 默认要求：明确输出风格、确认规则和质量要求。
- 可用技能：选择稳定、可复用的方法。
- 可用工具：只绑定完成任务真正需要的工具。
- 安全边界：敏感动作交给审批策略控制。

不要把临时账号、密码、token 或一次性任务内容写进智能体说明。

## 常见场景

### 给团队准备通用入口

适合创建一个“通用智能体”，用于日常问答、资料整理、轻量分析和方案建议。它应该保持简单，不要绑定过多专用规则。

### 给固定工作准备入口

如果团队经常做同一类工作，例如周报、客户回复、合同审查、投研整理，可以创建一个专用入口，并关联对应 Skill。

### 给需要精调的用户准备扩展入口

普通用户也可以在权限允许时创建个人入口，用于尝试新的技能或工具。大多数用户可以直接使用管理员准备好的公共入口，不需要自己配置；熟悉后再精调个人入口，稳定后由管理员整理为公共入口。

## 注意事项

- 智能体是入口，不是越多越好。
- 名称和说明要面向使用者，不要堆技术术语。
- 公共入口调整会影响多人，修改前应确认用途。
- 工具和密钥的使用仍受权限、连接和审批策略限制。
- 未开放或未稳定的入口，不建议作为普通用户主入口。

## 常见问题

### 智能体和 Skill 有什么区别？

智能体是工作入口，Skill 是完成某类任务的方法。一个智能体可以关联多个 Skill。

### 普通用户需要创建智能体吗？

通常不需要。管理员准备公共入口后，普通用户直接使用即可。如果用户已经熟悉自己的固定工作流程，也可以在个人范围内创建或调整智能体。

### 为什么我只能看到公共智能体？

可能当前账号没有个人智能体，或权限只允许查看公共入口。以页面显示为准。
