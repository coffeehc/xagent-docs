# xAgent 工作区文件：材料、结果与下载产物

## 适用对象

本文适合需要上传文件、查看产物、预览报告或复用任务材料的用户。

## 这是什么

工作区是 xAgent 保存任务材料和结果的地方。你上传的文件、任务生成的 Markdown、CSV、HTML 报告和中间结果，通常都会出现在工作区文件中。

xAgent 会按用户和任务范围隔离文件。日常使用时，只需要关注页面里能看到的文件。

## 什么时候使用

下面情况需要关注工作区：

- 任务需要读取文件、表格、PDF、文档或图片。
- 任务要生成可下载的文件。
- 需要预览 HTML 报告或其他结果。
- 需要在后续会话中继续引用已有材料。
- 需要确认某个结果是否已经保存。

## 基本用法

### 上传文件

1. 在 Agent会话 或工作区相关入口上传文件。
2. 等待文件上传完成。
3. 在任务里说明要处理哪个文件。
4. 如果文件很多，说明处理范围和优先级。

示例：

```text
请阅读我刚上传的三份 PDF，只整理和“采购流程”有关的内容，输出摘要和原文证据。
```

### 引用文件

引用文件时，优先使用页面中能看到的文件名或路径。例如：

```text
请处理工作区里的 customer_feedback.xlsx，找出反馈最多的 5 类问题。
```

不要写本机桌面路径，也不要写服务器内部路径。xAgent 只能处理当前工作区和授权范围内的文件。

### 查看结果

任务完成后，先看会话回复。如果回复中提到已生成文件，再进入工作区文件查看。

常见结果包括：

| 文件类型 | 常见用途 |
| --- | --- |
| Markdown | 总结、清单、会议纪要、轻量报告 |
| CSV / Excel | 表格结果、结构化数据、统计明细 |
| HTML | 带样式、图表或完整排版的报告 |
| PDF | 需要固定版式或外发的报告 |
| 图片 | 图表、截图、视觉结果 |

### 预览和下载

如果结果是 HTML、Markdown 或图片，优先使用预览入口确认内容。如果需要转发、存档或继续处理，再下载文件。

预览时重点检查：

- 标题和内容是否正确。
- 表格或图表是否完整。
- 中文是否显示正常。
- 是否包含不应公开的敏感信息。
- 文件是否来自当前任务，而不是旧结果。

### 继续使用已有产物

工作区文件可以继续作为下一步任务材料。例如：

```text
请基于刚生成的 summary.md，再整理一版给管理层看的 5 分钟汇报提纲。
```

如果文件名相似，建议明确说明文件名，避免使用错材料。

## 文件安全

使用工作区时注意：

- 不要上传不该进入系统的敏感文件。
- 外发前检查报告里是否包含个人信息、密钥、内部地址或客户数据。
- 删除、覆盖、发送等动作可能需要审批。
- 需要长期使用的材料，应使用稳定文件名和目录。
- 临时文件不一定适合作为正式交付结果。

## 常见问题

### 工作区是否等同于本机任意目录？

不是。工作区有明确可见性和权限边界。用户只能看到授权范围内的文件。

### 上传后的文件会自动被处理吗？

不一定。上传只是提供材料。你还需要在任务里说明“请阅读这个文件”或“请分析这个表格”。

### 为什么找不到生成的文件？

可能任务只在会话里回复了结果，没有保存文件。可以继续要求“请把结果保存为 Markdown 文件”。

### 可以把工作区文件发给外部系统吗？

可以，但通常需要工具、连接和审批。发送前应检查内容是否适合外发。
