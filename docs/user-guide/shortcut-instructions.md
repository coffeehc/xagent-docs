# 快捷指令

> 状态：实验性能力。统一的定向和对象引用格式仍在逐步接入，具体以当前页面提示为准。

快捷指令用于执行明确、固定的会话操作。它们不是发给 Agent 的普通任务，也不需要模型判断用户意图。

## 怎么使用

在 Agent 会话的输入框中输入 `/`，页面会显示当前会话可用的命令。选择命令并发送即可。

命令通常需要单独发送，不要在后面追加普通任务说明。例如：

```text
/compress
```

如果命令不适用于当前会话，或者会话正在执行任务、等待确认，页面会提示当前不能执行。

## 当前可用命令

| 命令 | 适用范围 | 作用 |
| --- | --- | --- |
| `/refresh_messages` | 主会话、子会话 | 重新同步当前页面中的会话消息 |
| `/delete` | 仅子会话 | 删除当前子会话并从会话列表移除 |
| `/clear-history` | 仅主会话 | 清理主会话当前可见历史，从新的上下文起点继续 |
| `/compress` | 主会话、子会话 | 手工压缩当前会话上下文 |

页面只会推荐当前会话可用的命令。例如，主会话不会显示 `/delete`，子会话不会显示 `/clear-history`。

## 重新同步消息

```text
/refresh_messages
```

当消息显示不完整、页面状态没有及时更新，或者重新连接后想刷新当前会话时，可以使用该命令。

它只重新同步页面消息，不会重新执行任务，也不会删除服务端记录。

## 删除子会话

```text
/delete
```

该命令仅用于子会话。执行后，子会话会从当前会话列表中移除，不能用于删除主会话。

删除前应先确认需要保留的结果已经回传到主会话，或者相关文件已经保存到合适位置。

## 清理主会话历史

```text
/clear-history
```

该命令仅用于主会话。它会把较早消息移出当前可见历史，并让后续任务从新的上下文起点继续。

需要注意：

- 这不是安全擦除功能，不代表底层历史记录已经被物理删除。
- 工作区文件、Skill、Tool、密钥配置和用户记忆不会因此删除。
- 需要保留的重要结论，应先保存为文件或整理到新的任务说明中。

也可以使用 `/clear_history` 或 `/clear history`，系统会按同一个命令处理。页面默认展示标准写法 `/clear-history`。

## 手工压缩上下文

```text
/compress
```

当会话已经很长，希望在保留当前任务主线的情况下减少上下文占用时，可以手工触发压缩。

压缩会整理当前目标、进度、约束、关键事实和产物引用。它不会删除工作区文件，也不等同于清理历史。压缩过程可能需要等待一段时间，并会使用模型能力。

如果当前没有足够内容可压缩，页面会直接提示。会话正在执行任务或等待审批时，应先处理当前状态，再执行压缩。

## 定向发送和对象引用

xAgent 正在统一下面两种快捷格式：

```text
@{session:会话ID} 继续处理这个任务
#{approval:审批ID} 同意
```

- `@{session:会话ID}` 表示把后面的消息或命令发送给指定会话。
- `#{类型:ID}` 表示在消息中引用一个明确对象，例如审批、文件或会话。

这部分仍处于实验阶段，并不是所有入口和对象类型都已经开放。当前 `v0.0.3.beta` 的审批通知也可能继续显示 `#审批号 同意` 这种兼容格式。实际使用时应优先采用页面、连接器通知或候选菜单自动生成的格式，不建议手工猜测对象 ID。

## 注意事项

- 未知的 `/命令` 会直接报错，不会作为普通任务交给 Agent。
- 命令只处理明确操作，不要把普通任务说明写在命令后面。
- 会话运行中或等待确认时，部分命令不可用。
- `/clear-history` 不是数据销毁工具。
- 实验性定向和引用格式应以当前页面实际提示为准。
