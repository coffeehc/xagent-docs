# Agent会话

## 适用对象

本文适合每天使用 xAgent 提交任务、查看过程、补充材料和拿结果的用户。

## 这是什么

Agent会话 是 xAgent 最主要的工作页面。你可以把它理解为“和工作助手协作的地方”：在这里输入需求、上传材料、查看执行过程、确认风险动作，并继续追问或修正结果。

![xAgent Agent会话](/img/manual/xagent-agent-session.png)

## 什么时候使用

下面这些事情都适合从 Agent会话 开始：

- 让 xAgent 阅读文档、网页、表格或已有材料。
- 让 xAgent 生成总结、报告、清单、回复、方案或数据结果。
- 需要跨多轮沟通，逐步补充要求。
- 需要查看工具执行过程和最终结果。
- 需要确认某些动作是否允许继续执行。

## 页面怎么看

左侧是会话列表，用于切换不同任务。中间是当前会话的消息和执行过程。底部是输入区，用来发送新要求、补充信息或上传附件。

页面顶部可能显示 token、工具调用次数、上下文等运行信息。普通用户不需要理解这些数字，只需要知道它们用于帮助管理员判断任务消耗和执行状态。

## 基本用法

### 新建或进入会话

1. 打开左侧菜单的 Agent会话。
2. 如果已有会话，点击左侧会话卡片进入。
3. 如果要开始新任务，点击新建入口。
4. 在输入框里说明要完成什么。
5. 如果有文件，先上传附件，再说明要处理哪个文件。

### 写第一条消息

一条好消息通常包含四类信息：

| 信息 | 写法示例 |
| --- | --- |
| 目标 | 请整理这份会议记录 |
| 材料 | 使用我刚上传的 `meeting.md` |
| 要求 | 按“决策 / 待办 / 风险”分组 |
| 结果 | 保存为 Markdown，并在会话里给我摘要 |

可以直接写成：

```text
请阅读我刚上传的 meeting.md，整理会议决策、待办事项、负责人和风险点，保存为 Markdown，并在会话里给我一个摘要。
```

### 查看执行过程

会话中可能出现几类内容：

| 内容 | 代表什么 | 你需要做什么 |
| --- | --- | --- |
| 普通回复 | xAgent 给出的说明或结果 | 阅读结果，继续追问或确认 |
| 工具调用 | xAgent 正在读取文件、访问网页、生成文件或发送请求 | 等待完成，必要时确认 |
| 工具观察 | 工具返回的结果 | 普通用户通常只看最终整理结果 |
| 失败提示 | 某一步没有成功 | 根据提示补材料、授权或调整要求 |
| 等待确认 | 有风险动作需要人工同意 | 确认内容无误后再允许继续 |

如果某一步没有完成，不要马上新开一个会话。优先在原会话里补充说明，例如“改用这个文件”“不要发送，只生成草稿”“把结果保存到工作区”。

### 上传和引用文件

涉及文件时，建议按下面方式描述：

- “请阅读我刚上传的文件。”
- “请处理工作区里的 `report.csv`。”
- “请把结果保存为 `summary.md`。”
- “生成 HTML 报告后给我预览入口。”

不要写本机任意路径，也不要把服务器内部路径当成可访问文件。用户只需要引用页面里能看到的文件名或工作区路径。

### 继续修正结果

如果结果方向正确但还不够好，可以继续在同一个会话里补充：

- “保留原文中的关键证据。”
- “输出改成表格。”
- “增加一列优先级。”
- “不要发送邮件，先给我草稿。”
- “把最终版保存到工作区。”

同一个会话会保留上下文，更适合连续调整。

## 常见场景

### 整理资料

```text
请阅读我上传的三份材料，整理共同结论、分歧点和需要确认的问题，结果用 Markdown 表格输出。
```

### 生成报告

```text
请基于工作区中的销售数据生成一份周报，包含核心指标、异常说明、原因猜测和下周建议，并保存为 HTML 报告。
```

### 处理外部消息

```text
请查看最近一条客户消息，先给出回复草稿，不要直接发送。
```

### 分阶段完成

```text
请先列出处理计划，等我确认后再开始读取文件和生成结果。
```

## 注意事项

- 涉及发送、删除、外部接口、敏感文件时，先确认内容再继续。
- 不要把密码、token、验证码直接写进会话。
- 如果任务材料很多，先说明范围和交付格式。
- 如果需要稳定复用一类流程，可以让管理员沉淀成 Skill。
- 如果会话已经很长，可以让 xAgent 总结当前状态，再继续下一步。

## 常见问题

### 为什么看到了工具调用？

这表示 xAgent 正在执行具体动作，例如读文件、查网页、生成文件或调用外部系统。普通用户重点看最终结果和是否需要确认。

### 为什么任务没有继续？

常见原因是缺少材料、等待审批、工具不可用、外部系统未连接，或任务描述不够明确。先看会话里的提示，再补充信息。

### 可以中途换要求吗？

可以。建议直接说明新的要求，例如“前面的结果保留，但输出格式改成表格”。

### 什么时候开新会话？

当目标已经变成另一件事，或不希望延续当前上下文时，再新建会话。
