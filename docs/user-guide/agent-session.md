# xAgent Agent 会话：提交任务、文件与审批

## 适用对象

本文适合每天使用 xAgent 提交任务、查看过程、补充材料和拿结果的用户。

## 这是什么

Agent会话 是 xAgent 最主要的工作页面。你可以把它理解为“和工作助手协作的地方”：在这里输入需求、上传材料、查看执行过程、确认风险动作，并继续追问或修正结果。

这不是普通聊天窗口。它同时承担任务提交、上下文延续、工具执行、文件处理、审批确认、子会话协作和任务微调等职责，所以需要单独说明。

![xAgent Agent 会话页面，显示会话列表、任务执行过程和消息输入框](/img/manual/xagent-agent-session.png)

## 什么时候使用

下面这些事情都适合从 Agent会话 开始：

- 让 xAgent 阅读文档、网页、表格或已有材料。
- 让 xAgent 生成总结、报告、清单、回复、方案或数据结果。
- 需要跨多轮沟通，逐步补充要求。
- 需要上传文件、查看生成文件或继续引用会话产物。
- 需要查看工具执行过程和最终结果。
- 需要确认某些动作是否允许继续执行。
- 需要把任务拆给子会话，或回到主会话汇总进展。
- 需要在任务过程中微调提示词、Skill、Tool、模型或密钥。

## 页面结构

Agent会话 页面主要分为几块：

| 区域 | 作用 | 常见操作 |
| --- | --- | --- |
| 左侧会话列表 | 切换主会话和子会话 | 搜索会话、选择会话、刷新列表、右键修改标题或复制会话标识 |
| 顶部工具栏 | 查看当前会话运行信息 | 查看 token、上下文、工具调用次数，切换 Tool 调用显示，打开高级设置或会话文件 |
| 中间时间线 | 查看任务过程和结果 | 阅读用户消息、xAgent 回复、思考过程、工具调用、审批和状态提示 |
| 底部输入区 | 继续提交要求 | 输入消息、上传附件、发送、终止当前执行 |
| 会话文件面板 | 查看当前会话文件 | 刷新会话文件、预览或下载产物 |
| 高级设置 | 微调当前会话能力 | 调整 Agent Prompt、模型策略、密钥、常驻 Skill 和常驻 Tool |

普通用户通常只需要关注左侧会话列表、中间时间线和底部输入区。高阶用户需要让当前任务达到更好状态时，再进入高级设置做微调。

## 主会话与子会话

Agent会话里会看到两类会话：

| 类型 | 可以理解为 | 适合做什么 |
| --- | --- | --- |
| 主会话 | 总负责人 | 直接沟通目标、拆分任务、创建子会话、汇总各会话进展 |
| 子会话 | 专项执行者 | 处理从主会话拆出来的独立任务，例如定时采集、资料研究、邮件检查、某个专项报告 |

主会话和子会话都可以直接输入消息。区别在于主会话更适合统筹和汇总，子会话更适合处理独立目标。

进入子会话后，顶部可能出现“回主会话”按钮。需要回到总入口继续沟通或汇总时，点击它即可。

## 基本用法

### 新建或进入会话

1. 打开左侧菜单的 Agent会话。
2. 如果已有会话，点击左侧会话卡片进入。
3. 如果要开始新任务，点击新建入口或在主会话里说明要创建新的智能体会话。
4. 在输入框里说明要完成什么。
5. 如果有文件，先上传附件，再说明要处理哪个文件。

### 写第一条消息

一条好消息通常包含四类信息：

| 信息 | 写法示例 |
| --- | --- |
| 目标 | 请整理这份会议记录 |
| 材料 | 使用我刚上传的 `meeting.md` |
| 要求 | 按“决策 / 待办 / 风险”分组 |
| 结果 | 保存为 Markdown，并在会话里给我摘要 |

可以直接写成：

```text
请阅读我刚上传的 meeting.md，整理会议决策、待办事项、负责人和风险点，保存为 Markdown，并在会话里给我一个摘要。
```

### 继续修正结果

如果结果方向正确但还不够好，可以继续在同一个会话里补充：

- “保留原文中的关键证据。”
- “输出改成表格。”
- “增加一列优先级。”
- “不要发送邮件，先给我草稿。”
- “把最终版保存到工作区。”

同一个会话会保留上下文，更适合连续调整。只有目标已经变成另一件事，或不希望延续当前上下文时，才建议新建会话。

## 时间线怎么看

会话中可能出现几类内容：

| 内容 | 代表什么 | 你需要做什么 |
| --- | --- | --- |
| 用户消息 | 你发送的任务、补充说明或附件 | 确认目标和材料是否写清楚 |
| xAgent 回复 | xAgent 给出的说明、结果或阶段性结论 | 阅读结果，继续追问或确认 |
| 思考过程 | xAgent 的任务拆解或执行思路 | 只作为过程参考，不必逐字检查 |
| 工具调用 | xAgent 正在读取文件、访问网页、生成文件或调用外部系统 | 等待完成，必要时查看详情 |
| 工具观察 | 工具返回的结果 | 普通用户通常只看最终整理结果 |
| 等待审批 | 有风险动作需要人工同意 | 看清动作、对象和影响后再允许或拒绝 |
| 等待确认 | 系统需要你补充字段、选择项或确认方案 | 按页面要求补齐信息 |
| 状态提示 | 运行中、重试、上下文压缩、连接恢复等状态 | 了解当前是否还能继续输入或需要等待 |
| 失败提示 | 某一步没有成功 | 根据提示补材料、授权、调整要求或重试 |

顶部有“显示Tool调用”开关。如果关闭，已完成的工具调用会被隐藏，时间线更适合普通阅读；如果需要排查过程或确认某一步做了什么，可以打开。

## 输入区与附件

底部输入区用于继续给当前会话发送消息。常见用法：

- 直接输入任务目标。
- 点击加号上传文件或图片。
- 粘贴截图、文本或文件。
- 在任务运行中补充说明。
- 必要时终止当前执行。

涉及文件时，建议按下面方式描述：

- “请阅读我刚上传的文件。”
- “请处理工作区里的 `report.csv`。”
- “请把结果保存为 `summary.md`。”
- “生成 HTML 报告后给我预览入口。”

不要写本机任意路径，也不要把服务器内部路径当成可访问文件。用户只需要引用页面里能看到的文件名或工作区路径。

如果附件仍在处理中，页面会阻止发送。等附件处理完成后再提交，避免 xAgent 拿不到文件内容。

## 工具调用与审批

xAgent 完成任务时可能调用工具，例如读写文件、访问网页、处理表格、发送邮件、调用连接器或创建子会话。

普通用户不需要记工具名，只要说明想完成的动作。真正涉及风险时，系统会通过审批或确认流程拦住。

需要重点确认的动作包括：

- 发送邮件、微信或其他外部消息。
- 删除文件或修改重要数据。
- 访问外部系统或调用内部接口。
- 创建子会话或长期触发器。
- 使用密钥、连接器或高风险工具。

看到审批时，先看清楚它要做什么、会影响哪个对象、是否会向外部发送内容。确认无误后再允许继续；如果不确定，可以拒绝，然后在会话里补充说明。

## 会话文件

顶部的会话文件入口用于查看当前会话相关文件。它适合：

- 找到刚上传的材料。
- 查看工具生成的中间文件。
- 预览 Markdown、HTML、表格或报告。
- 下载最终产物。

如果任务生成了文件，但你在消息里没看到入口，可以打开会话文件面板刷新一次。

## 高级设置与任务微调

高级设置用于配置当前会话的能力，不是每个用户都必须使用。它适合熟悉系统的用户在任务过程中做微调。

常见设置包括：

| 设置 | 用途 | 什么时候调整 |
| --- | --- | --- |
| Agent Prompt | 修改当前会话的工作方式和注意事项 | 当前任务需要更明确的角色、边界或输出风格 |
| 模型与 Provider Policy | 指定当前会话使用的模型或请求策略 | 当前模型不适合阅读、推理、写作、视觉或工具调用时 |
| 密钥 | 选择当前会话允许引用的密钥 key | 工具需要访问邮箱、API 或外部系统时 |
| 常驻 Skill | 让某些 Skill 长期留在当前会话上下文 | 长任务必须固定某套工作方法时 |
| 常驻 Tool | 让某些 Tool 长期出现在当前会话工具列表 | 当前任务长期依赖固定工具时 |

这里的密钥只选择 key，真实密钥值不会进入模型或消息历史。

任务微调的目标不是让用户承担配置负担，而是在当前任务需要更高质量时，让高阶用户可以逐步调整上下文、能力和模型，直到任务达到更好的完成状态。

## 上下文与长会话

顶部会显示上下文和 token 信息，用于提示当前会话的上下文使用情况。普通用户不需要理解每个数字，只要知道：

- 会话越长，系统越需要管理上下文。
- xAgent 可能进行上下文压缩。
- 压缩过程中会冻结部分会话操作。
- 压缩完成后可以继续执行。

如果会话已经很长，可以让 xAgent 先总结当前状态，再继续下一步。

## 会话命令

输入区支持少量命令，适合排障或管理当前会话：

| 命令 | 作用 | 注意 |
| --- | --- | --- |
| `/refresh_messages` | 重新同步当前会话消息 | 消息显示异常时使用 |
| `/compress` | 手工压缩当前会话上下文 | 长会话继续执行前使用 |
| `/clear-history` | 清空主会话历史消息 | 仅主会话可用 |
| `/delete` | 删除当前子会话 | 仅子会话可用 |

如果当前会话仍在运行或等待确认，命令可能暂时不可用。

## 常见场景

### 整理资料

```text
请阅读我上传的三份材料，整理共同结论、分歧点和需要确认的问题，结果用 Markdown 表格输出。
```

### 生成报告

```text
请基于工作区中的销售数据生成一份周报，包含核心指标、异常说明、原因猜测和下周建议，并保存为 HTML 报告。
```

### 处理外部消息

```text
请查看最近一条客户消息，先给出回复草稿，不要直接发送。
```

### 分阶段完成

```text
请先列出处理计划，等我确认后再开始读取文件和生成结果。
```

### 微调当前任务

```text
这个任务更偏正式报告，请切换到更适合长文写作的模型，并按“背景 / 发现 / 建议 / 风险”结构重新组织。
```

## 注意事项

- 涉及发送、删除、外部接口、敏感文件时，先确认内容再继续。
- 不要把密码、token、验证码直接写进会话。
- 如果任务材料很多，先说明范围和交付格式。
- 如果需要稳定复用一类流程，可以沉淀成 Skill。
- 如果当前任务需要更高质量，可以在同一会话里逐步微调，不要频繁换新会话。
- 如果会话已经很长，可以让 xAgent 总结当前状态，再继续下一步。

## 常见问题

### 为什么看到了工具调用？

这表示 xAgent 正在执行具体动作，例如读文件、查网页、生成文件或调用外部系统。普通用户重点看最终结果和是否需要确认。

### 为什么任务没有继续？

常见原因是缺少材料、等待审批、等待你补充信息、工具不可用、外部系统未连接，或任务描述不够明确。先看会话里的提示，再补充信息。

### 可以中途换要求吗？

可以。建议直接说明新的要求，例如“前面的结果保留，但输出格式改成表格”。

### 什么时候开新会话？

当目标已经变成另一件事，或不希望延续当前上下文时，再新建会话。只是在同一个目标内调整格式、补充材料、换模型或换 Skill，不需要新开会话。

### 高级设置改错了怎么办？

先回到会话里说明问题，或重新打开高级设置恢复原来的提示词、模型、Skill、Tool 和密钥选择。系统级配置仍由管理员维护。
