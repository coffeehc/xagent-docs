# 会话

## 适用对象

本文适合需要创建、进入、切换或观察会话的用户。

## 这是什么

会话是一次持续工作的上下文。它保存当前消息、任务状态、可见能力、上传附件、工作区产物和运行过程中的提示。

## 什么时候使用

每次和 xAgent 讨论一个目标、项目、文件或问题时，都应该在一个合适的会话中进行。不要把无关任务混在同一个会话里。

## 基本用法

使用会话时建议：

1. 从 Sessions 页面进入已有会话，或使用主会话开始。
2. 给会话一个能识别目标的名称，便于后续查找。
3. 在同一个会话中持续补充材料、确认工具调用、追问结果。
4. 把同一项目的文件和产物保存在当前会话的工作区路径下。
5. 如果目标发生明显变化，创建新会话或切换到更合适的会话。

会话页面中需要重点关注：

- 当前回复和历史消息。
- 运行状态、等待确认、失败提示。
- 上传附件和处理状态。
- 生成的文件、报告和可预览产物。
- 当前可用工具、技能、Secret 和模型配置。

## 常见问题

### 会话是否只是聊天记录？

不是。聊天记录只是入口。会话还包含任务上下文、能力选择、文件产物和执行状态。

### 会话是否保存所有长期事实？

不应该。需要跨会话复用的偏好、决策或背景应进入记忆或其他明确位置。

### 会话是否可以被子任务执行单元复用？

长任务或拆分任务可能创建独立执行单元。普通用户只需要关注主会话中的状态和回传结果。
