# 外部连接

> 状态：实验性能力，页面和字段可能变化。

## 适用对象

本文适合需要把微信、邮件、企业系统或第三方服务接入 xAgent 的用户和管理员。

## 这是什么

外部连接用于让 xAgent 在授权范围内处理外部系统里的消息、数据或动作。普通用户可以把它理解为“把外部账号接进来”，管理员则负责准备连接器和系统级配置。

使用手册不要求普通用户理解连接协议。只需要确认连接是否可用、账号是否授权、任务里要处理什么对象。

## 什么时候使用

下面情况需要外部连接：

- 读取微信、邮件或其他消息。
- 根据外部消息生成回复。
- 查询企业系统中的数据。
- 将结果发送到外部渠道。
- 让外部事件自动触发 xAgent 任务。

如果只是处理本地上传文件，不一定需要外部连接。

## 页面入口

外部连接相关入口可能包括：

| 菜单 | 用途 |
| --- | --- |
| 我的连接 | 当前用户自己的外部账号或授权 |
| Connector 管理 | 管理员维护连接器和连接状态 |
| MCP 配置 | 管理外部能力服务 |
| 我的 MCP | 当前用户自己的外部能力服务 |
| 密钥管理 | 保存个人密钥引用 |

普通用户优先看 我的连接。管理员再看 Connector 管理 和 MCP 配置。

## 基本用法

### 普通用户连接账号

1. 打开 我的连接。
2. 找到要连接的外部系统。
3. 按页面提示完成授权或绑定。
4. 确认连接状态正常。
5. 回到 Agent会话，说明要处理的外部任务。

示例：

```text
请查看最近一条客户微信消息，先整理回复草稿，不要直接发送。
```

### 管理员准备连接器

管理员接入外部系统时，建议确认：

- 连接器是否可用。
- 需要哪些账号授权。
- 会提供哪些工具或动作。
- 哪些动作需要审批。
- 出错时用户能否看懂提示。
- 是否会接触敏感数据。

外部连接应按最小权限准备。不要为了方便给过大的系统权限。

### 在任务中描述外部对象

普通用户用自然语言描述即可：

- “最近一条客户消息”
- “今天收到的新邮件”
- “某个客户的订单”
- “这个群里的待办”
- “我绑定的默认邮箱”

如果页面要求选择账号或渠道，按页面提示选择。

## 安全注意

- 不要把真实密码、token、验证码写进会话。
- 发送消息前建议先生成草稿。
- 外部系统的数据权限仍由外部系统和连接器控制。
- 连接异常时，不要反复提交同一任务，先检查连接状态。
- 涉及客户、财务、合同等敏感信息时，按团队安全策略处理。

## 常见问题

### 外部连接和工具有什么关系？

外部连接提供账号和通道，工具执行具体动作。例如连接邮箱后，工具才能读取或发送邮件。

### 我需要把外部系统密码告诉 xAgent 吗？

不需要。按页面授权流程操作，不要把密码、token 或验证码写进任务消息。

### 为什么连接成功但任务仍然失败？

可能是工具未启用、权限不足、审批未通过、外部系统不可用，或任务描述没有明确对象。

### 外部连接是否就是 Skill？

不是。外部连接负责接入外部系统，Skill 负责沉淀任务方法。
