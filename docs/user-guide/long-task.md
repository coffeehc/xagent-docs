# 长任务

> 状态：实验性能力，接口可能变化。

## 适用对象

本文适合需要理解长时间执行、拆分执行和独立推进机制的读者。

## 这是什么

长任务是不能在一次短交互内稳定完成的 Task。它可能需要计划、阶段性结果、外部 Tool 调用、Workspace 产物和持续状态管理。

## 什么时候使用

当任务需要多步骤执行、长时间等待、跨环境处理或独立子任务推进时，可以考虑长任务机制。

## 基本用法

当前阶段不写固定 API。描述长任务时应关注：

- 当前目标和完成条件。
- 已完成阶段和待执行阶段。
- 是否需要 SubAgent。
- 是否存在人工确认、失败恢复或结果归档。

## 常见问题

### 所有长任务都需要 SubAgent 吗？

不一定。SubAgent 是长任务的一种执行组织方式，不应写成必然行为。

### 长任务状态是否等同于 Session 状态？

不等同。具体状态归属需要以实现为准。

### 为什么不写命令示例？

当前接口未稳定，示例命令容易误导。
