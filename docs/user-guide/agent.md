# 智能体

## 适用对象

本文适合需要选择、配置或理解智能体行为的用户。

## 这是什么

智能体是在会话中执行任务的工作主体。它会根据当前会话上下文、可见工具、技能、记忆、模型配置和用户要求决定下一步。

在团队使用中，常见做法是管理员先配置好通用智能体或专用智能体，普通用户直接选择并使用；只有当工作流稳定、角色需要长期复用时，才需要新增或调整智能体。

## 主 Agent 和子 Agent

xAgent 中可以从使用角度把 Agent 分成两类：主 Agent 和子 Agent。

主 Agent 是用户在 Agent会话 中主要面对的工作助手。它负责理解用户目标、维护当前会话方向、安排工具和技能、判断是否需要确认，并最终把结果交付给用户。

子 Agent 是主 Agent 为复杂任务临时拆分出来的子任务执行者。它通常只负责一个明确的小范围目标，例如阅读一组材料、核对某类数据、整理某个章节或做一次独立检查。子 Agent 的结果会回到主 Agent，由主 Agent 汇总、取舍和继续推进。

| 对比项 | 主 Agent | 子 Agent |
| --- | --- | --- |
| 面向对象 | 直接面向用户 | 面向主 Agent 分配的子任务 |
| 任务范围 | 负责整体目标和最终交付 | 负责局部目标和阶段结果 |
| 使用方式 | 用户在会话中直接交互 | 通常由系统在复杂任务中自动使用 |
| 结果归属 | 对最终结果负责 | 向主 Agent 返回子任务结果 |
| 适用场景 | 普通问答、文件处理、报告生成、外部任务协作 | 多材料分析、长任务拆分、并行检查、阶段性研究 |

普通用户可以把主 Agent 理解为“总负责人”，把子 Agent 理解为“临时分工的助手”。子 Agent 不会绕过当前会话、工作区、工具权限或审批策略。

## 什么时候使用

当你需要固定一个工作角色、输出习惯、模型选择或能力组合时，需要关注智能体配置。

## 基本用法

使用智能体时建议：

- 首次使用先用默认智能体，不急于创建复杂角色。
- 需要长期复用时，再配置专门智能体或专家会话。
- 管理员可以先把常用模型、工具、技能和安全策略配置到对应智能体上。
- 角色说明写长期职责，不写一次性任务材料。
- 工具、技能、Secret 和模型配置要和任务类型匹配。
- 会话执行过程中如需更换模型，可以优先调整当前会话的模型配置，而不是重新开始任务。
- 如果输出风格不稳定，优先补充明确的格式要求。

智能体不应被当成“拥有所有事实”的对象。当前文件、历史、权限、外部系统登录态和长期记忆都有各自边界。

## 常见问题

### 智能体是否直接保存长期记忆？

不直接保存。记忆是独立能力，智能体只在上下文中使用被投影出来的内容。

### 智能体是否直接管理外部系统登录态？

不管理。外部系统登录态属于外部连接或目标系统。

### 智能体是否总是自动执行所有工具？

不会。工具调用需要遵守当前会话、能力选择、权限和审批规则。

### 子 Agent 是否需要用户手动管理？

通常不需要。用户只需要描述任务目标和验收要求。是否拆分子任务，以及如何汇总子任务结果，由主 Agent 和系统执行过程处理。
