# MCP 与连接器有什么区别

MCP 和连接器都能扩展 xAgent 的能力，但它们解决的问题不同。一个简单判断是：

- 任务执行时需要按需调用某项外部能力，优先考虑 MCP。
- 外部系统需要把账号、消息或事件主动送入 xAgent，优先考虑连接器。

## 核心区别

| 对比项 | MCP | 连接器 |
| --- | --- | --- |
| 在 xAgent 中的主要作用 | 为任务提供可按需调用的外部工具和服务 | 接入外部账号、消息、通道、事件和连接器工具 |
| 典型方向 | xAgent 在任务过程中调用外部能力 | 外部系统可以主动把消息或事件投递到 xAgent |
| 是否关注用户账号和登录状态 | 通常不以账号接入为主要目标 | 通常需要处理扫码、OAuth、授权或其他登录流程 |
| 常见场景 | 查询业务数据、访问知识服务、调用办公或研发工具 | 微信、邮件、IM、企业系统消息入口、第三方服务连接 |
| 普通用户常见入口 | 在任务中使用已配置的能力 | 在“我的连接”中绑定个人外部账号 |
| 管理员常见入口 | 配置 MCP 服务和可用工具 | 在“Connector 管理”中接入服务、检查 Connector Card 和 health |

以上是 xAgent 当前的使用分工，并不是对外部协议能力的绝对限制。实际选择应以任务是否需要账号、消息入口和主动事件为准。

## 当前运行路径

MCP 服务接入后，xAgent 建立连接并把服务提供的能力注册为可调用工具。会话在执行任务时按需发现、加载和调用这些工具。MCP 的主线是“会话主动调用能力”，不负责 xAgent 的用户账号绑定和主动消息入口。

连接器是独立的外部桥接服务，负责目标系统协议、登录态和通道状态。外部消息进入连接器后，连接器可以主动推送给 xAgent；xAgent 再把消息转换为会话事件，按明确的会话引用确定目标，未指定时进入当前用户主会话。会话事件只在同一用户范围内投递，不支持跨用户会话通信。

## 什么时候用 MCP

当外部系统更像一项“需要时调用的能力”时，适合使用 MCP。例如：

- 查询一个内部数据服务。
- 调用已提供 API 的知识、办公或研发工具。
- 在完成任务时读取、写入或处理某类业务数据。
- 让 Agent 在需要时发现并加载特定工具，而不是在会话开始时加载全部能力。

对普通用户而言，通常不需要先理解 MCP 的配置。管理员或进阶用户把能力接入、测试并开放后，用户只需要在任务中说明目标即可。

## 什么时候用连接器

当外部系统需要作为 xAgent 的主动入口，或需要保留用户自己的账号与通道状态时，适合使用连接器。例如：

- 微信收到新消息后，连接器主动把消息投递到 xAgent。
- 用户通过扫码、OAuth 或授权绑定自己的外部账号。
- 邮件、IM 或企业系统事件需要触发后续任务。
- 需要在授权范围内向外部系统发送消息、媒体或执行动作。

连接器不仅能让 xAgent 调用外部工具，也能把外部系统中的事件带进来。用户通常在“我的连接”完成账号绑定；管理员在“Connector 管理”维护系统级连接器服务。具体操作见[连接器](/docs/user-guide/connector)。

## 按场景判断

| 你的需求 | 更适合的方式 | 原因 |
| --- | --- | --- |
| 任务需要查询一个固定的外部服务 | MCP | 会话在需要时调用能力即可，不一定需要账号和消息入口。 |
| 希望微信消息进来后自动进入任务 | 连接器 | 需要外部系统主动接收消息并投递事件。 |
| 用户需要绑定自己的邮箱、IM 或其他账号 | 连接器 | 需要维护用户级认证和连接状态。 |
| 已有企业 API，只希望 Agent 在任务中调用 | MCP | 更接近按需调用的工具服务。 |
| 企业系统既有用户身份、消息事件，也有可执行动作 | 连接器优先评估 | 先处理账号、通道和事件入口，再确认要暴露哪些工具和动作。 |

## 示例

### 示例一：读取业务数据

你希望 Agent 在整理报告时查询一套内部数据接口。这个接口不需要接收外部消息，也不需要用户扫码登录。此时可把它作为 MCP 能力接入，让 Agent 在任务中按需查询。

### 示例二：从微信发起任务

你希望用户在微信中发送消息后，由 xAgent 自动接收、创建或推进任务。此时需要连接器。连接器负责接收消息、维护授权和通道状态，并把事件送入 xAgent。

### 示例三：接入企业内部系统

如果只需要让 Agent 查询或调用一组稳定 API，可以优先考虑 MCP；如果还需要账号绑定、消息推送、登录流程或主动事件，则更适合按连接器方式设计。

## 安全边界

- 不论使用 MCP 还是连接器，外部系统的账号权限仍由外部系统自身控制，xAgent 不替代原有权限体系。
- 不要把真实密码、Token 或验证码写进会话、提示词或任务材料。
- 涉及发送消息、修改数据、审批或敏感信息访问时，应配置[审批策略](/docs/user-guide/approval-policy)。
- 连接器服务需要保存外部登录态时，应使用独立运行用户和数据目录，并避免直接暴露管理端口到公网。
