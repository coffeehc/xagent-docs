# 使用 xAgent 完成第一个任务

## 适用对象

本文适合准备用 xAgent 执行第一个任务的读者。

## 这是什么

任务是你希望 xAgent 完成的目标。第一个任务不需要复杂，只要目标明确、材料清楚、交付方式可判断即可。

## 什么时候使用

当你已经能进入 Agent会话，并希望验证 xAgent 是否能真正完成一件事时，按本文开始。

## 基本用法

描述任务时，可以先使用下面的结构化模板：

```text
目标：
输入材料：
必须遵守的约束：
输出格式：
结果保存位置：
```

不需要机械填写每一项。任务越复杂，越应该明确材料范围、限制条件和结果形式。

一个适合新手的任务可以这样写：

```text
请把这段需求整理成待办清单，按“必须 / 应该 / 可选”分组，最后给出一个 Markdown 表格。
```

如果任务涉及文件，先上传文件或把文件放到工作区，再说明要处理哪个文件：

```text
请阅读我刚上传的会议记录，整理决策、待办、负责人和截止时间，并保存为 Markdown。
```

提交后关注三件事：

1. xAgent 是否请求补充材料。
2. 是否出现需要确认的工具调用或权限请求。
3. 最终结果是直接回复，还是保存到了工作区产物中。

如果结果不符合预期，继续在同一个页面中补充要求，例如“保留原始表述”“增加风险列”“把结果保存成 CSV”。

## 完整案例：整理会议记录

### 输入

上传一份名为 `meeting-notes.md` 的会议记录，其中包含讨论内容、决定、负责人和时间要求。

### 任务

```text
请读取 meeting-notes.md，提取会议决定、待办事项、负责人、截止时间和风险项。
生成一份便于阅读的 Markdown 总结，并把待办事项同时保存为 CSV。
信息不明确时请标记为“待确认”，不要自行补充。
```

### 执行时需要确认

1. xAgent 是否读取了正确的文件。
2. 是否保留了原文中能够确认的负责人和日期。
3. 信息缺失时是否明确标记，而不是自行猜测。
4. 最终是否同时生成 Markdown 和 CSV 文件。

### 输出

工作区中可以得到：

```text
meeting-summary.md
meeting-actions.csv
```

`meeting-summary.md` 的内容可以类似：

```markdown
## 会议决定

- 新版本先向测试用户开放，再根据反馈扩大范围。

## 待办事项

| 待办 | 负责人 | 截止时间 | 风险 |
| --- | --- | --- | --- |
| 整理测试问题清单 | 张三 | 7 月 18 日 | 部分问题缺少复现步骤 |
| 确认发布范围 | 待确认 | 待确认 | 需要负责人补充 |
```

这个案例的验收标准很明确：来源文件正确、关键信息可追溯、缺失内容没有被编造，并且两个结果文件都已保存。

## 常见问题

### 任务是否一定会拆成很多步骤？

不一定。简单任务通常会直接完成。只有材料很多、步骤复杂或需要中途确认时，才可能分阶段推进。

### 任务是否等同于一次用户消息？

不等同。一次用户消息可以创建新任务，也可以只是补充材料、修正要求或确认某个步骤。

### 一个好任务应该包含什么？

至少包含目标、材料、约束和交付格式。越需要工具或文件处理，越应该写清楚输入来源和结果形式。
