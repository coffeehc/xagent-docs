# 第一个任务

## 适用对象

本文适合准备用 xAgent 执行第一个任务的读者。

## 这是什么

任务是你希望 xAgent 完成的目标。第一个任务不需要复杂，只要目标明确、材料清楚、交付方式可判断即可。

## 什么时候使用

当你已经能进入 Agent会话，并希望验证 xAgent 是否能真正完成一件事时，按本文开始。

## 基本用法

一个适合新手的任务可以这样写：

```text
请把这段需求整理成待办清单，按“必须 / 应该 / 可选”分组，最后给出一个 Markdown 表格。
```

如果任务涉及文件，先上传文件或把文件放到工作区，再说明要处理哪个文件：

```text
请阅读我刚上传的会议记录，整理决策、待办、负责人和截止时间，并保存为 Markdown。
```

提交后关注三件事：

1. xAgent 是否请求补充材料。
2. 是否出现需要确认的工具调用或权限请求。
3. 最终结果是直接回复，还是保存到了工作区产物中。

如果结果不符合预期，继续在同一个页面中补充要求，例如“保留原始表述”“增加风险列”“把结果保存成 CSV”。

## 常见问题

### 任务是否一定会拆成很多步骤？

不一定。简单任务通常会直接完成。只有材料很多、步骤复杂或需要中途确认时，才可能分阶段推进。

### 任务是否等同于一次用户消息？

不等同。一次用户消息可以创建新任务，也可以只是补充材料、修正要求或确认某个步骤。

### 一个好任务应该包含什么？

至少包含目标、材料、约束和交付格式。越需要工具或文件处理，越应该写清楚输入来源和结果形式。
