# xAgent 产品介绍与核心能力

## 适用对象

本文适合第一次了解 xAgent 的使用者、管理员和评估者。

## 这是什么

xAgent 是面向团队与公司的多用户智能工作门户。它把常用工作入口、文件、工具、技能、外部连接和安全策略放在一个网页系统里，让团队可以快速搭建可直接使用的专用工作助手。

当前版本为 `v0.0.2.beta`，仍处于测试版阶段。它适合部署体验、功能验证、场景评估和社区反馈，不应按已经完全稳定的企业正式版理解。

特别说明：测试版中很多功能还不完善。如果在使用或阅读文档时遇到问题，可以直接在 GitHub 提交 issue：[https://github.com/coffeehc/xagent-docs/issues](https://github.com/coffeehc/xagent-docs/issues)。

日常使用时，用户通过网页说明目标、上传材料、确认风险动作并查看结果。管理员负责提前准备常用场景、模型、工具、技能、外部连接和安全策略，降低普通用户的配置负担。

xAgent 为完成任务而生，不是命令行项目，也不只是聊天界面。它当然可以被当成聊天工具或陪伴搭子使用，但它的核心性格更像一个认真推进工作的助手：理解目标、处理材料、调用能力、保存产物，并在需要时请求确认。如果你只想找一个轻松闲聊的搭子，它未必是最讨喜的选择。

普通用户应从 Agent会话、任务、文件、工具、技能和外部连接这些功能入口开始使用。

## 服务端部署

xAgent 部署在服务器端，而不是安装在每个用户的个人电脑上。用户通过 Web 页面访问，也可以通过扩展的连接器接入微信、邮件、企业系统或其他入口来下达任务。

部署时只需要运行一个二进制文件。Linux 环境下建议注册为 systemd 服务，方便开机启动、后台运行和日志管理。当前主要支持 macOS 和 Linux。Windows 下目前没有测试环境，也没有经过测试，不建议尝试。

这种方式带来几个直接好处：

- 不需要在每台电脑上都安装一个智能体。任务在服务器端运行，执行过程中不要求用户电脑一直开机。
- 用户可以通过 Web 使用，也可以通过 IM 工具远程下达任务，例如通过微信等入口触发任务处理。
- 任务文件集中存储在服务端，并按用户做物理隔离，用户只能看到自己的工作区和授权范围内的文件。
- 如果团队使用本地模型或自有模型服务，可以让任务数据留在自己的环境中，做到更高程度的数据私有化。

## 安全治理

xAgent 面向多用户任务执行，因此安全治理不是只保护一个密钥，而是同时处理文件、密钥、工具动作、外部连接和会话边界。

当前版本重点做了几类收敛：

- 工作区隔离：任务文件集中保存在服务端，并按用户隔离。用户只能看到自己的工作区和授权范围内的文件，不能直接浏览其他用户的任务文件。
- 密钥隔离：xAgent 对密钥做独立管理。用户和管理员可以保存外部系统、模型服务或工具需要使用的密钥，工具在配置和调用时只使用占位符引用密钥。
- 密钥不进模型：只有在 xAgent 内部真正调用工具时，系统才会把占位符替换为实际密钥值。模型只看到任务目标、工具说明和必要参数，不需要看到密钥本身。用户也不要把密钥明文写进任务消息或提示词。
- 敏感动作审批：发送消息、删除或覆盖文件、调用外部系统、访问敏感数据等动作，可以通过审批策略要求人工确认，或由管理员直接拒绝。
- 外部连接边界：连接器和 MCP 可以接入微信、邮件、企业系统或第三方服务，但外部系统本身的账号权限、登录态和数据权限仍应由对应系统控制。xAgent 不应该替代外部系统的权限体系。
- 会话通讯边界：同一用户下的 session 可以通过会话事件协作，但当前不支持跨用户 session 通讯，避免把一个用户的上下文、文件或任务状态带到另一个用户范围内。
- 私有化部署：xAgent 暂无 SaaS 发布计划，更推荐部署在用户自己的服务器和数据环境中。配合本地模型或自有模型服务时，可以进一步减少业务数据离开团队环境的机会。

这些能力不能替代组织内部的账号管理、备份、审计、合规和最小权限配置。xAgent 的目标是在智能体执行任务时把常见风险入口收口，让用户和管理员能看见、能控制、能审批。

## 关于用户的说明

xAgent 当前的用户模型是扁平的。系统内的用户没有组织结构里的上下级关系，也不承担“谁向谁汇报、谁审批谁工作”的企业流程。

这样设计是因为 xAgent 的定位是智能助手基座，而不是 OA、ERP、CRM 或企业权限中心。把复杂组织权限放进智能体基座里并不合适；更重要的是，数据权限也不应该交给大模型判断。模型可以帮助理解任务、组织材料和调用能力，但不应该成为决定“某个用户能否访问某份业务数据”的权限裁判。

xAgent 负责管理自己的边界，例如用户工作区隔离、会话范围、密钥引用、工具可用性和敏感动作审批。外部业务系统的数据权限，应由外部系统自己控制：

- 通过 MCP 调用外部系统时，访问范围由 MCP 背后的系统、账号和授权决定。
- 通过连接器接入微信、邮件、企业系统或第三方服务时，登录态、账号权限和数据范围由对应连接器和目标系统决定。
- 如果企业已有统一身份、角色、部门、数据权限和审计体系，应通过定制集成或连接器接入，而不是让 xAgent 在内部重新实现一套组织权限系统。

简单说，xAgent 管好智能体执行任务时自己的边界；业务数据属于哪个系统，就由哪个系统负责授权和审计。

## 和常见智能体有什么不同

常见的智能体产品往往是一个聊天入口，或一个围绕固定场景设计的专用助手。xAgent 更像一个智能体基座门户：它不是只提供一个固定助手，而是让团队在同一套系统里组织多个专用工作入口。普通用户可以直接使用准备好的入口，熟悉后也可以在个人范围内精调自己的 Skill、工具、连接和智能体配置。

| 对比项 | 常见聊天助手 | 单一专用智能体 | xAgent |
| --- | --- | --- | --- |
| 主要目标 | 回答问题、陪聊或生成内容 | 解决一个固定场景 | 围绕任务组织多个可用、可管、可扩展的工作助手 |
| 使用方式 | 用户直接对话 | 用户进入固定场景使用 | 管理员准备能力，用户按任务入口直接使用 |
| 能力扩展 | 依赖产品内置能力 | 通常围绕单一业务定制 | 通过 Skill、工具、MCP、连接器和审批策略扩展 |
| 结果形态 | 多数是文本回复 | 取决于专用场景 | 回复、文件、报告、外部动作、长期任务和工作区产物 |
| 安全治理 | 通常较弱或隐藏 | 按场景处理 | 用户、会话、工作区、工具和敏感动作都有边界 |

因此，xAgent 不是让用户从零配置每一次对话，而是让团队先准备好通用能力，再把常见工作沉淀成可以反复使用的专用智能体。

## 如何实现别的智能体功能

如果要在 xAgent 里实现新的智能体能力，通常不需要重新开发一套系统。可以按下面路径逐步完成：

1. 先明确这个智能体要完成什么任务，例如合同审查、客户回复、数据分析、投研整理或内部知识问答。
2. 创建或选择一个专用智能体入口，让用户知道应该从哪里开始。
3. 把稳定的处理方法沉淀为 Skill，让同类任务不必每次重新说明。
4. 如果任务需要执行动作，启用对应工具，例如读写文件、处理表格、访问网页或发送消息。
5. 如果任务需要连接外部系统，配置连接器或 MCP。
6. 如果涉及删除、发送、外部请求或敏感数据，配置审批策略。
7. 如果需要接入企业内部系统、统一身份或专属流程，再做定制开发和集成。

xAgent 也可以辅助生产 Skill。用户可以把一套人工流程、历史任务或现有说明交给 xAgent，让它整理成 Skill 草稿，再进一步测试、优化和发布。对于稳定高频的工作，xAgent 可以配合专门的 Skill 持续生成测试用例、检查执行效果、修订说明并发布新版本，让技能逐步变得更好用。

## Skill 与 Tool

截至 2026 年 7 月 7 日，当前版本内置约 49 个 Skill，覆盖报告生成、资料研究、文档理解、写作编辑、合同与合规、财务分析、客户支持、销售运营、项目管理、招聘培训、个人效率和 Skill 创建等常见工作场景。后续审计和迭代过程中，内置 Skill 还可能继续增加。

截至 2026 年 7 月 7 日，当前版本内置约 79 个 Tool ID，覆盖会话协作、任务计划、文件读写、Excel、PDF、SQLite、网页抓取、HTTP 请求、邮件、触发器、Skill/Tool 动态发现与加载等能力。其中一部分是系统内部或按需加载工具，普通用户实际看到的数量会受账号权限、外部连接、MCP 配置和管理员策略影响。后续审计和迭代过程中，内置 Tool 也可能继续增加。

## 连接器

连接器用于把微信、邮件、企业系统、第三方服务或其他外部入口接入 xAgent。它不是单纯的接口配置，而是把外部系统里的消息、账号状态、可执行动作和事件入口，以可管理的方式投递到 xAgent。

连接器常见用途包括：

- 接收外部消息，例如微信、邮件或企业系统通知。
- 把外部消息转换为会话事件，发送到对应的智能体会话。
- 提供外部系统工具，例如读取消息、查询数据、发送回复或更新状态。
- 管理外部系统授权，让用户通过自己的账号连接外部服务。
- 配合审批策略控制敏感动作，例如发送消息、修改外部数据或调用内部系统。

连接器和 MCP 都可以扩展 xAgent 的外部能力，但侧重点不同：

| 对比项 | 连接器 | MCP |
| --- | --- | --- |
| 主要作用 | 接入外部系统、账号、消息和事件入口 | 提供一组可被调用的外部工具或服务能力 |
| 是否能主动推送 | 可以。连接器可以主动向 xAgent 发送消息或事件 | 通常由 xAgent 在任务中按需调用 |
| 典型场景 | 微信收到消息后，连接器接收消息并主动推给 xAgent | xAgent 需要查询资料、调用工具或访问外部能力时再调用 |
| 用户感知 | 更像“把外部入口接进来” | 更像“给 xAgent 增加一组可用工具” |

简单说，MCP 更偏“xAgent 需要时去调用能力”，连接器则可以“外部系统有消息时主动把任务推给 xAgent”。

普通用户通常只需要在“我的连接”里完成授权，然后在 Agent会话 中说明要处理什么。管理员则负责准备连接器、配置可用能力和安全策略。

## 动态能力

xAgent 的每个会话都具备动态发现与加载能力。会话启动时，系统会默认加载一组核心工具，例如 `skill_find`、`skill_load`、`tool_find`、`tool_load`。这些工具不会把所有 Skill 和 Tool 一次性塞进上下文，而是让智能体在任务执行过程中按需查找和加载。

当任务进行到某一步，发现当前上下文里缺少合适的 Skill 或 Tool 时，对应提示词会指导智能体先寻找可用能力，再加载需要的能力继续执行。例如，用户让 xAgent 分析表格、生成报告或连接外部系统时，如果当前会话还没有加载相关能力，xAgent 可以先发现，再按需加载。

这个设计有两个目的：

- 节省上下文：会话不需要一开始就携带全部 Skill 和 Tool，只加载当前任务真正需要的部分。
- 自动扩展能力：智能体在遇到新需求时，可以主动补齐所需 Skill 或 Tool，更好地完成任务。

普通用户不需要记住这些工具名。只要描述任务目标，xAgent 会在执行过程中尽量发现和加载合适能力。

## 会话事件总线与会话间通讯

xAgent 内置会话事件队列。外部接口、触发器、连接器和其他会话都可以发起会话事件，事件会通过队列发送到对应的智能体会话中。

xAgent 支持同一用户下的会话间通讯。会话之间不是直接共享全部上下文，而是通过会话事件传递信息。目前提供两类事件：通知和协助。

| 事件 | 用途 | 是否进入模型上下文 |
| --- | --- | --- |
| 通知 | 在界面上提醒另一个会话有新的状态、结果或信息 | 不进入模型上下文 |
| 协助 | 请求另一个会话参与处理任务，并激活协助任务 | 会进入模型上下文 |

通知事件只在 UI 上显示，用于让用户看到会话之间的状态变化。协助事件会进入模型上下文，并触发对应的协助任务，适合在复杂工作中让不同会话配合完成一件事。

会话事件不只服务于会话之间的协作，也可以作为外部任务进入 xAgent 的统一入口。例如，触发器可以在指定时间发送任务事件，连接器可以把外部消息转换为会话事件，外部接口也可以把业务系统里的任务投递到指定会话。

目前，单个用户下的所有 session 可以互相通信，但不支持跨用户 session 通讯。跨用户会话通讯会带来权限、数据泄露和上下文污染等安全风险，因此 xAgent 在这个能力上保持收敛。

## 自进化能力

xAgent 具备自进化能力，但这不是默认打开的功能。团队可以通过明确提示词，或通过专门的自进化 Skill，让 xAgent 对某类 Skill 进行持续优化、测试和发布。

这件事需要克制使用。就目前大模型的执行逻辑而言，自进化是双刃剑：如果目标、评价标准和测试样例是正确的，它可能沿着正确方向持续改进；如果一开始选错方向，或评价标准不清楚，它也可能在错误路径上越走越远。

因此，xAgent 更推荐把自进化用于边界清楚、可测试、可回滚的技能改进，而不是让系统无约束地修改自己的全部行为。需要精调能力的用户和管理员应该先定义目标、样例、验收标准和发布流程，再让自进化能力介入。

## 开放与热切换

xAgent 的很多能力都可以在系统内配置和调整。用户可以在权限允许的范围内调整提示词、Skill、工具、外部连接和审批策略，管理员可以维护模型和系统级配置，而不是每改一次配置都重新部署一套服务。

多数设置支持热切换。任务执行过程中，也可以根据实际情况切换提示词、切换 Skill 或切换模型。例如，一个任务前半段可以使用更适合资料整理的模型，后半段切换到更适合写报告的模型；也可以在发现当前 Skill 不适合时，调整或替换 Skill 后继续推进。

这种自由度的目标不是让普通用户承担配置负担，而是让大多数用户可以直接使用准备好的能力；熟悉系统的用户可以继续精调个人能力，管理员再把稳定结果沉淀给团队直接使用。

## 任务微调

这里说的“微调”不是训练模型，而是在当前任务执行过程中细调目标、上下文和可用能力，让任务尽量达到最佳状态。

普通用户可以直接使用管理员准备好的智能体、Skill、Tool、连接器和模型配置，不需要理解底层细节。高阶用户在熟悉系统后，可以围绕当前任务做更细的调整：

- 补充或收窄任务目标、背景材料和验收标准。
- 切换或加载更适合当前任务的 Skill、Tool 或连接器能力。
- 调整当前会话的提示词或智能体入口，让执行方式更贴近任务。
- 在任务过程中切换模型，分别利用不同模型在阅读、推理、写作、工具调用等方面的优势。
- 补充需要的密钥、MCP、连接器授权或个人审批规则，让任务可以继续执行。
- 将有效的调整沉淀为个人 Skill、个人智能体或团队公共配置，减少下次重复调试。

因此，xAgent 的使用不是一次性把所有配置定死。普通用户可以无感使用已有能力，高阶用户可以在任务执行中逐步微调，把当前任务推到更好的完成状态。

## 发布定位

当前版本是 `v0.0.2.beta` 测试版，面向免费二进制发布。目标是让用户先完成部署、进入产品、体验核心流程，并判断 xAgent 是否适合自己的智能体工作场景。

免费二进制测试版适合了解标准能力、验证常见任务流程和评估团队落地方式。它不等同于稳定企业正式版、免费企业定制、免费托管服务、免费内部系统集成或长期支持承诺。涉及企业级身份认证、内部系统连接、权限审计、专用外部连接和安全策略时，通常需要结合实际环境做定制开发和集成。

当前发布方式是免费二进制发布，不等同于源码开源。xAgent 对开源保持开放态度，未来是否开放源码会结合产品成熟度、社区参与、安全边界和商业可持续性综合判断。持续维护一个可用于真实任务的智能体基座需要长期投入，因此 xAgent 会在开放使用、社区协作和可持续发展之间保持平衡。

目前 xAgent 暂无 SaaS 发布计划。xAgent 更倾向于让用户把系统部署在自己的服务器和数据环境中，而不是把任务文件、业务数据和外部系统连接存储到官方平台。用户自行部署后，数据存储、访问控制、备份、审计和保密责任由部署方自行承担，官方不承担用户业务数据的托管和保密责任。

## 数据库

当前版本默认使用内置 SQLite 数据库。xAgent 面向团队和公司自部署使用，如果默认要求对接 PostgreSQL 等大型数据库，会明显增加部署、维护和排障成本。

因此，当前版本优先选择 SQLite 来降低上手门槛，让用户可以更快完成部署和体验。xAgent 具备后续升级到 PostgreSQL 等数据库的能力，适合在规模、并发、审计或运维要求提高后再按实际需要升级。

## 使用方式

xAgent 的推荐使用方式是“管理员准备好，用户直接用，有固定流程再沉淀”。

- 管理员准备用户、模型、常用工作入口、技能、工具、外部连接和安全策略。
- 普通用户进入 Agent会话，描述要完成的工作，也可以直接使用 Skill、Tool、连接器、触发器和智能体配置。大多数时候不需要配置，熟悉后可以把稳定流程保存成 Skill 或精调个人入口，减少重复说明和手工配置。

这种方式适合多人共用同一套基础能力：普通用户不必理解模型参数、工具清单或底层协议，只需要说明任务目标和交付要求。

## 什么时候使用

当任务具备下面任意特征时，可以考虑使用 xAgent：

- 需要多轮沟通和持续上下文。
- 需要读取、生成或整理文件。
- 需要调用工具、接口或外部系统。
- 需要把过程结果保存到工作区。
- 需要长期保留偏好、决策或项目背景。
- 需要把复杂工作分阶段推进。
- 希望由管理员统一配置基础能力，多个用户直接复用。
- 希望把模型能力、安全策略、文件隔离和外部系统连接放在同一个入口里管理。

## 能力特点

- 专用工作助手：按场景准备不同入口，用户进入后直接描述任务。
- 文件工作区：上传材料、保存中间结果、预览报告和下载交付产物。
- 工具和技能：工具负责执行具体动作，技能沉淀可复用的任务方法。
- Skill 生产：可以辅助整理、测试、优化和发布 Skill，让团队把稳定流程沉淀为可复用能力。
- 动态能力：会话可以按需发现和加载 Skill 与 Tool，减少上下文占用，并在任务中自动补齐所需能力。
- 连接器：接入微信、邮件、企业系统或第三方服务，让外部消息、账号状态和可执行动作进入会话。
- 服务端执行：任务在服务器端运行，用户电脑不需要一直在线，远程入口也可以提交任务。
- 集中存储：任务文件和产物集中保存在服务端工作区，并按用户隔离。
- 多用户安全：用户、会话、工作区和工具动作都有明确边界，敏感动作可以通过策略和审批控制。
- 密钥隔离：工具使用密钥占位符，实际密钥只在内部工具调用时替换，不直接交给模型。
- 基础记忆：xAgent 具备基础记忆能力，记忆在用户范围内共享，用于保留偏好、背景和长期任务信息。
- 知识能力扩展：当前没有内置知识库，但可以通过 Skill + MCP 接入外部知识、检索服务或团队已有资料系统。
- 会话事件总线：外部接口、触发器、连接器和会话都可以发起会话事件，并通过队列投递到目标智能体会话。
- 长任务支持：适合需要持续推进、阶段拆分、等待确认或产物归档的任务。
- 模型选择：会话可以根据任务需要调整模型配置，不必因为模型变化重新开始整个工作。
- 任务微调：提示词、Skill、Tool、模型和上下文可以在任务过程中按需调整，让当前任务尽量达到最佳状态。
- 私有化模型：配合本地模型或自有模型服务，可以让数据留在团队自己的环境中。
- 克制的自进化：可以通过提示词或自进化 Skill 持续优化特定能力，但建议放在清晰边界和测试流程内使用。

## 管理员和维护者说明

普通用户不需要理解技术实现。管理员或维护者只需要先知道下面几点：

- xAgent 基于 Go 开发，运行在服务端。
- 用户通过 Web 或 IM 连接器访问 xAgent，不需要在每台个人电脑上安装独立智能体。
- 当前版本默认使用内置 SQLite 数据库，以降低部署难度；后续可按需求升级到 PostgreSQL 等数据库。
- 外部能力可以通过 MCP 或连接器接入。
- xAgent 在底层文件系统之上提供工作区隔离，用户只能看到授权范围内的文件。
- 任务文件和产物集中存储在服务端，可以结合本地模型或自有模型服务提高数据私有化程度。
- xAgent 具备基础记忆能力，记忆按用户范围共享；用户也可以通过 MCP 扩展自己的记忆系统。
- 当前没有内置知识库，知识类能力可以通过 Skill + MCP 组合接入外部知识库、检索系统或团队已有资料系统。
- 会话事件通过内置队列投递到目标智能体会话，目前支持通知和协助；出于安全考虑，不支持跨用户 session 通讯。
- 密钥由 xAgent 独立管理，工具通过占位符引用密钥，实际值只在内部工具调用时替换。
- 技能可以在系统内管理、编辑和加载，用于沉淀稳定任务方法。
- 会话默认具备 Skill 和 Tool 的发现与加载能力，用于按需扩展任务执行能力并节省上下文。
- 提示词、Skill、模型等设置可以热切换，方便在任务执行过程中调整策略。
- 技能可以通过 xAgent 辅助生产、测试、优化和发布，但自进化流程应有明确目标、验收标准和回滚方式。

## 后续计划

xAgent 后续会继续围绕任务完成能力演进，具体优先级会结合社区反馈调整。当前方向包括：

- 基于项目的智能体 Team 功能：后续会开发面向项目的智能体团队协作能力。当前版本也可以通过变通方式实现类似效果，例如让一个会话监控或协助另一个会话。
- 小脑能力：后续会集成一个小模型，聚焦摘要、路由、意图预理解、OCR、向量化等处理。主要目标是减少上下文占用，并持续监控会话执行稳定性，提高任务完成度。
- 记忆体强化升级：现有基础记忆能力会继续增强，具体形态需要结合社区反馈来确定。
- 知识库开发：后续会开发更直接的知识库能力。在此之前，可以通过 Skill + MCP 接入外部知识库或检索系统。
- 周边生态开发：继续补充更多连接器、MCP 和场景能力，例如接入视频生成框架、飞书等，让 xAgent 能连接更多实际工作入口和外部生产工具。
- 其他方向：根据真实使用场景和社区反馈继续调整。

## 基本用法

推荐从下面路径开始：

1. 如果还没有部署服务，先阅读 [服务端安装](/docs/deployment/server-install)，把 xAgent 服务和基础依赖准备好。
2. 通过 [功能导览与菜单入口](/docs/user-guide/menu-overview) 了解主要功能和控制台入口。
3. 对照左侧菜单找到要打开的页面。
4. 如果你是普通用户，直接进入 Agent会话 并说明要完成的工作。
5. 如果你是管理员，先准备常用工具、技能、外部连接和安全策略。
6. 如果你有固定重复流程，再阅读 [创建 / 更新 Skill](/docs/getting-started/create-skill)，把流程保存下来并持续优化。

## 常见问题

### xAgent 是一个聊天机器人吗？

不是。聊天只是入口之一，xAgent 更关注把任务做完：读取材料、生成文件、等待确认、接入外部系统、归档结果。它可以聊天，但产品目标不是陪聊，而是完成工作。

### xAgent 需要安装在每台电脑上吗？

不需要。xAgent 部署在服务器端，用户通过 Web 或扩展的连接器访问。任务执行过程中不要求用户电脑一直开机，用户也可以通过微信等 IM 入口远程提交任务。

### 任务文件保存在哪里？

任务文件和产物集中保存在服务端工作区，并按用户做隔离。用户只能看到自己的工作区和授权范围内的文件。

### 数据可以完全私有化吗？

如果使用本地模型或自有模型服务，可以让任务数据留在团队自己的环境中。是否能做到完全私有化，还取决于模型、外部连接、部署方式和管理员配置。

### 密钥会被发送给模型提供商吗？

正常不会。xAgent 的工具配置使用密钥占位符，实际密钥只在内部调用工具时替换。模型提供商不需要看到密钥实际值。用户也不要把密钥、token 或密码明文写进任务消息。

### xAgent 有记忆能力吗？

有。xAgent 具备基础记忆能力，记忆在用户范围内共享，用于保存偏好、长期背景和反复使用的信息。用户也可以通过 MCP 扩展自己的记忆系统。后续也会考虑开放记忆接口，让第三方记忆系统可以和 xAgent 直通。

### xAgent 有内置知识库吗？

目前没有内置知识库。知识类能力可以通过 Skill + MCP 扩展：Skill 负责描述如何使用知识和组织回答，MCP 负责连接外部知识库、检索服务或团队已有资料系统。

### 会话事件从哪里来？

会话事件可以来自外部接口、触发器、连接器，也可以来自其他会话。xAgent 会通过内置事件队列把事件投递到对应的智能体会话中。

### 连接器是什么？

连接器负责把微信、邮件、企业系统或第三方服务接入 xAgent。它可以接收外部消息、提供外部工具、管理授权状态，并把外部事件投递到智能体会话中。普通用户主要完成授权和使用，管理员负责准备连接器和安全策略。

### 会话之间可以互相通信吗？

可以，但仅限同一用户下的会话。xAgent 通过会话事件实现会话间通讯，目前有通知和协助两类事件。通知只在界面显示，不进入模型上下文；协助会进入模型上下文，并激活协助任务。跨用户 session 通讯暂不支持，因为这会带来安全风险。

### xAgent 是单一专用智能体吗？

不是。xAgent 可以承载多个专用智能体入口。管理员可以为不同工作场景准备不同的智能体、Skill、工具、外部连接和安全策略，普通用户进入对应入口后直接使用。

### xAgent 能帮我做新的专用智能体吗？

可以。通常先明确任务场景，再准备专用智能体入口、Skill、工具、外部连接和审批策略。如果只是沉淀一套固定工作方法，可以先让 xAgent 辅助生成 Skill 草稿，再测试、优化和发布。

### 每个会话都要预先加载所有 Skill 和 Tool 吗？

不需要。xAgent 会话默认具备发现和加载 Skill、Tool 的核心能力。任务执行过程中如果发现当前能力不够，会尝试查找并加载合适的 Skill 或 Tool。这样既节省上下文，也让智能体能按任务需要扩展自身能力。

### xAgent 会自动自我进化吗？

默认不会。xAgent 可以通过提示词或自进化 Skill 对特定技能进行持续优化，但这需要明确目标、测试样例、验收标准和发布流程。自进化很有价值，但也有风险，所以 xAgent 对这件事保持克制。

### 可以自己调整配置吗？

可以。模型、提示词、Skill、工具、外部连接和审批策略等配置可以在系统内维护。很多设置支持热切换，任务执行过程中也可以根据需要调整提示词、Skill 或模型。

### 普通用户需要自己配置工具和技能吗？

通常不需要。管理员可以先配置通用场景、专用智能体、工具、技能和外部连接，普通用户直接在会话里使用。

### 免费二进制版本意味着什么？

免费二进制版本目前是 `v0.0.2.beta` 测试版，是了解和评估 xAgent 的入口。用户可以先部署标准版本，体验任务提交、文件工作区、工具、技能和外部连接等核心能力。

免费二进制发布不等同于源码开源。xAgent 未来会视产品成熟度、社区反馈、安全边界和商业可持续性，评估是否开放源码或开放更多生态协作方式。

如果需要企业内部系统接入、统一身份认证、复杂权限、审计合规、专属连接器或深度业务流程改造，通常应按实际需求进行定制集成。

### 商业版与免费版有什么区别？

商业版与免费版在产品功能上保持一致。目前差别主要体现在可使用的用户数限制上，不会把核心功能拆成免费版和商业版两套体验。

对于家庭部署、小型团队或早期场景验证，免费版已经可以完整体验 xAgent 的核心能力，部署和使用门槛很低。商业版更适合需要更多用户规模、配套支持或定制集成的团队与公司。

### xAgent 会提供官方 SaaS 吗？

目前暂无 SaaS 发布计划。xAgent 不希望用户把自己的任务文件、业务数据和外部系统连接存储到官方平台。自行部署时，数据安全、备份、权限和保密责任由部署方负责。

### xAgent 使用什么数据库？

当前版本默认使用内置 SQLite 数据库，目的是降低部署和维护难度。xAgent 具备升级到 PostgreSQL 等数据库的能力，适合在团队规模、并发、审计或运维要求提高后再按实际需要升级。

### xAgent 适合大型企业平台吗？

xAgent 可以作为企业智能工作门户和业务智能化入口的基础能力，用来承接统一任务入口、文件工作区、工具调用、技能沉淀和外部系统连接等场景。

如果用于大型公司或复杂组织，通常需要结合企业现有系统做定制集成，例如统一身份认证、权限体系、内部业务系统连接、审计要求、数据边界、安全策略、模型网关和专用外部连接。这类场景不是简单安装后立即覆盖全部企业流程，而是以 xAgent 为基础，根据企业已有基础设施逐步接入和扩展。

### xAgent 能替代所有业务系统吗？

不能。xAgent 负责组织任务入口和能力调用，外部系统的权限、登录态、业务数据和审计规则仍应由对应系统或外部连接管理。

### 这里会包含技术实现细节吗？

主线是使用手册。普通使用者优先阅读“开始使用”和“使用手册”；技术资料只作为维护和定制时的补充。
