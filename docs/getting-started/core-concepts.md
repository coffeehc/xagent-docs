# 术语说明

## 适用对象

本文适合维护文档、排查问题、做定制集成或阅读技术资料的人员。普通用户不需要先阅读本文。

## 这是什么

xAgent 技术资料中常见的术语包括：

| 概念 | 含义 |
| --- | --- |
| 会话（Session） | 一段连续工作上下文，包含消息、状态、可见能力和产物 |
| 智能体（Agent） | 在会话中理解目标、决定下一步并调用能力的工作主体 |
| 任务（Task） | 用户希望完成的目标，可以是一次简单请求，也可以拆成多阶段任务 |
| 工具（Tool） | 智能体可调用的具体能力，例如读写文件、查询数据、请求网页或发送邮件 |
| 技能（Skill） | 针对某类任务的可复用方法和工具使用说明 |
| 外部连接（Connector） | 把微信、邮件、企业系统或第三方 API 等外部系统接入 xAgent 的连接能力 |
| 工作区（Workspace） | 保存输入文件、中间结果、报告和可预览产物的空间 |
| 记忆（Memory） | 面向长期上下文的能力，用于保留有价值的偏好、决策和背景 |
| 子任务执行单元（SubAgent） | 面向长任务或拆分任务的独立执行单元 |
| 运行连接（RuntimeConnection） | 工具执行时可能依赖的运行环境，普通用户通常不直接操作 |

## 什么时候使用

当你需要和开发、实施或维护人员沟通系统边界时，可以查看本文。遇到固定名词时，以 [术语表](/docs/reference/glossary) 为准。

## 基本用法

理解一次系统处理过程时，可以按下面的顺序看：

```text
用户目标 -> 会话 -> 智能体 -> 工具 / 技能 / 外部连接 / 工作区 -> 结果与产物
```

如果任务变复杂，xAgent 可能继续使用记忆、子任务执行单元或触发器来保持上下文、拆分工作或定时执行。普通用户只需要按页面提示补充材料、确认动作和查看结果。

## 常见问题

### 技能和工具有什么区别？

工具是可以被调用的具体动作。技能是完成某类任务的方法说明，可能会指导智能体组合多个工具。

### 会话和任务是一回事吗？

不是。会话是持续工作上下文，任务是要完成的目标。一个会话里可以推进多个任务。

### 记忆是否等同于聊天历史？

不是。聊天历史属于当前会话，记忆面向长期上下文和跨任务复用。
