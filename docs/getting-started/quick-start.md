# 功能导览

## 适用对象

本文适合第一次使用 xAgent，希望先知道“有哪些功能、从哪里进入、能完成什么”的读者。

## 这是什么

功能导览按用户实际动作说明 xAgent 的主要功能。它不把 xAgent 当成命令行项目，也不要求普通用户理解 AI 技术名词。

xAgent 的使用主线是：管理员先准备常用入口和可用能力，普通用户直接进入网页提交任务、补充材料、确认动作并查看结果。

## 什么时候使用

第一次打开 xAgent、给新用户介绍功能、或不知道某件事该去哪个页面完成时，先阅读本文。

## 基本用法

### 使用路径

不同角色可以按下面路径开始：

- 普通用户：进入 Agent会话，说明要完成什么、材料在哪里、希望得到什么结果。
- 管理员：先准备账号、模型、常用工作入口、工具、技能、外部连接和安全策略。
- 进阶用户：把反复出现的固定流程保存成技能，减少重复说明。

第一次进入控制台时，可以先阅读 [菜单与功能入口](/docs/user-guide/menu-overview)，对照左侧菜单确认每个入口的用途。

### 首页

首页用于查看当前工作状态：最近会话、运行中的任务、等待确认、Token 和工具调用统计。日常使用时，可以先从首页判断哪些会话需要继续处理。

适合：

- 查看当前有哪些活跃任务。
- 进入某个会话继续处理。
- 发现等待审批或失败的任务。

### 提交任务

Agent会话 是最主要的工作入口。你可以在这里提出目标、上传材料、确认风险动作、查看回复和追踪产物。

适合：

- 让 xAgent 阅读文件、整理内容、生成报告。
- 围绕同一项目持续多轮沟通。
- 查看执行状态和结果。

### 查看文件和结果

工作区文件 用于保存上传材料、生成文件、中间结果和最终报告。涉及文件的任务，优先通过会话上传或在工作区选择文件。

适合：

- 上传文档、表格、PDF 或其他材料。
- 预览 Markdown、HTML、表格和报告。
- 下载或继续引用生成产物。

### 让系统执行动作

工具用于执行具体动作，例如读取文件、请求网页、处理表格、查询数据库、发送邮件或发布预览。普通用户不需要记工具名，只需要说明要完成的动作。

适合：

- “读取这个文件并总结。”
- “把结果保存成 Markdown。”
- “查询这个表格里的异常项。”
- “确认后发送邮件。”

### 复用固定流程

Skill 管理用于管理可复用的任务方法。它能帮助 xAgent 用更稳定的流程完成研究、写作、报告、会议纪要、合同审查等任务。

适合：

- 任务有固定步骤。
- 输出有固定质量要求。
- 需要组合多个工具。

### 接入外部消息和账号

我的连接、Connector 管理和 MCP 配置用于接入微信、邮件、企业系统或第三方服务。普通用户只需要关注账号是否已连接、任务是否已授权。

适合：

- 处理外部消息。
- 读取外部系统状态。
- 经确认后向外部渠道回复。

### 自动执行固定任务

触发器管理用于定时或由外部事件自动提交任务。

适合：

- 每天或每周生成固定报告。
- 到点提醒处理任务。
- 外部系统事件到达后自动进入会话处理。

### 第一次任务怎么写

第一条任务建议写清楚：

- 目标：要完成什么。
- 材料：需要读取的文件、链接或背景。
- 约束：输出格式、语言、范围、是否需要引用来源。
- 交付：希望得到文本回复、文件、报告、表格还是其他产物。

示例：

```text
请阅读我上传的需求文档，整理一份功能清单，按优先级分组，并把结果保存为 Markdown 报告。
```

### 查看结果和产物

任务执行过程中可以关注：

- 会话消息和状态提示。
- 工具调用、审批或失败提示。
- 工作区中生成的文件和预览链接。
- 需要人工补充的信息或确认。

## 常见问题

### 如果我还没有可访问的 xAgent 环境怎么办？

如果你只是使用者，先联系管理员获取访问地址和账号。普通用户不需要自己启动服务或修改配置。

### 为什么这里不先讲命令？

因为 xAgent 的主要使用入口是 Web 功能，不是命令行。命令只用于开发、部署和维护。

### 文档为什么只放少量完整截图？

目标用户更需要知道“这个菜单做什么”，而不是阅读大量技术说明。手册会保留少量完整界面截图，帮助识别页面位置；具体用法主要用文字说明。

### 任务不能完成怎么办？

先检查任务材料是否完整、模型是否可用、相关工具/技能/外部连接是否启用，以及会话里是否有等待确认或失败提示。
