# 第一个智能体

## 适用对象

本文适合第一次使用 xAgent，不确定是否需要创建自定义智能体的读者。

## 这是什么

智能体是在会话中理解目标、调用工具并反馈结果的工作主体。第一次使用时，不需要先设计复杂智能体；通常先使用默认会话和默认智能体，等任务稳定后再配置专用智能体或专家会话。

## 什么时候使用

当你需要让 xAgent 按某个长期角色、固定工具组合或特定工作方式执行任务时，再考虑配置智能体。

## 基本用法

第一次使用建议按下面顺序：

1. 先进入默认会话，直接描述任务。
2. 如果任务需要固定角色，例如“合同审查助手”“数据分析助手”，再进入智能体或专家相关页面配置角色说明。
3. 为智能体选择需要的工具、技能、模型和 Secret。
4. 回到会话中验证一次最小任务。
5. 如果效果稳定，再把它作为常用智能体使用。

写智能体角色说明时，优先描述长期稳定信息：

- 这个智能体负责什么类型的任务。
- 默认输出风格和交付格式。
- 常用工具、技能或外部连接。
- 不应处理的范围。

不要把当前一次任务的临时材料、当前文件路径、当前计划或一次性要求写进智能体角色说明。

## 常见问题

### 一定要先创建智能体才能开始吗？

不需要。大多数首次任务可以直接在默认会话中完成。

### 智能体和会话是什么关系？

智能体负责执行，会话承载上下文。一个智能体可以在会话中工作，但当前会话里的文件、历史、工具状态和用户补充仍属于会话上下文。

### 什么时候需要多个智能体？

当不同任务需要不同角色、工具组合、权限边界或输出习惯时，可以拆成多个智能体或专家会话。
