# 创建 / 更新 Skill

## 这是什么

Skill 是一套可复用的工作方法。把固定步骤、输入要求、输出格式和注意事项保存成 Skill 后，xAgent 在处理同类任务时就不需要每次重新解释流程。

创建或更新 Skill 的目标不是写代码，而是把一件重复工作说明清楚，并在真实使用中持续修正，让 xAgent 能稳定复用。

## 什么时候适合创建 Skill

下面这些场景适合创建 Skill：

- 每周、每天或经常重复的工作。
- 每次都有相似的材料、步骤和输出格式。
- 需要多人按同一套标准处理。
- 需要把个人经验沉淀下来，方便以后继续使用。
- 希望 xAgent 后续能持续优化、测试和发布这类工作方法。

如果只是一次性任务，直接在 Agent 会话里说明需求即可，不一定要创建 Skill。

## 什么时候适合更新 Skill

已经发布的 Skill 在使用过程中可能需要继续调整。下面这些情况适合更新：

- 输出格式不够稳定，需要补充字段、顺序或示例。
- 执行步骤不够清楚，导致模型容易走偏。
- 业务流程发生变化，需要更新规则、限制或确认动作。
- 新增了工具、MCP 或连接器能力，希望 Skill 能正确使用。
- 用户反馈某些场景处理不好，需要补充边界和反例。

## 创建前先想清楚

创建或更新 Skill 前，先准备下面几项内容：

| 内容 | 说明 |
| --- | --- |
| Skill 名称 | 让用户一眼知道它处理什么任务 |
| 适用场景 | 说明什么时候用，什么时候不适合用 |
| 用户需要提供什么 | 文件、链接、文字说明、目标对象或其他材料 |
| 处理步骤 | 按顺序说明 xAgent 应该怎么做 |
| 输出格式 | 报告、表格、清单、邮件草稿、Markdown 或其他格式 |
| 需要确认的动作 | 发送消息、删除文件、提交外部系统等动作要先确认 |
| 质量标准 | 什么样的结果算完成，哪些问题需要返工 |

不要把账号密码、API Key、验证码、客户隐私、临时文件路径或一次性信息写进 Skill。

## 创建入口

### 方式一：在 Skill 管理中创建或导入

适合已经准备好 Skill 内容，或者要导入已有 Skill 文件的用户。

1. 打开左侧菜单中的 Skill 管理。
2. 点击新增 Skill。
3. 填写名称、描述、适用场景和具体工作方法。
4. 保存后，在详情页检查内容是否清晰。
5. 回到 Agent 会话，用一个真实任务测试效果。
6. 根据测试结果继续修改。

也可以在 Skill 管理中导入已有 Skill 文件，再根据当前团队的使用方式做调整。

### 方式二：在主会话中描述 Skill 需求

适合只知道自己想要什么，还没有整理成 Skill 文档的用户。

可以在主 Agent 会话里这样说：

```text
我想创建一个用于合同审查的 Skill。
它需要检查付款、交付、违约、保密和争议解决条款，并输出风险清单和修改建议。
```

主会话会根据你的简单需求，自动创建一个用于 Skill 构建的智能体会话。进入这个会话后，xAgent 会先继续了解你的需求，例如适用场景、输入材料、输出格式、风险边界和确认动作。

需求确认清楚后，xAgent 会协助生成 Skill 草稿。草稿生成后，需要你确认内容是否符合预期；确认后，再继续完成 Skill 发布。发布之后，也可以继续在这个会话里根据测试结果优化 Skill。

整个过程中都是模型在协助你完成构建。因为创建 Skill 需要理解任务、抽象流程、检查边界并组织成可复用方法，建议使用智力更强、推理能力更好的模型。

### 方式三：直接创建 Skill 构建会话

如果你已经知道要专门构建一个 Skill，也可以直接通过智能体创建一个会话来完成。

进入 Skill 构建会话后，流程和主会话自动创建类似：

1. xAgent 先了解你的 Skill 需求。
2. 你补充任务背景、材料类型、输出要求和限制条件。
3. xAgent 生成 Skill 草稿。
4. 你确认或要求修改。
5. 确认后发布 Skill。
6. 发布后继续在当前会话中优化，或新建一个会话进行下一轮迭代。

这种方式适合比较复杂的 Skill，例如需要长期使用、多人复用、需要稳定输出格式，或者后续希望持续优化的工作方法。

## 更新 Skill

更新 Skill 时，可以在主会话或专门的 Skill 更新会话里说明你要更新哪个 Skill，以及希望改成什么样。

示例：

```text
我想更新“合同审查”这个 Skill。
现在它对付款条款的检查不够细，请增加付款节点、发票要求、逾期责任和验收条件的检查。
```

进入更新 Skill 的子会话后，xAgent 会自动把对应 Skill 拷贝一份到当前会话目录下，并把这份副本作为草稿。后续修改都围绕这份草稿进行，不会直接覆盖原始 Skill。

更新流程通常是：

1. 你描述要更新的 Skill 名称或用途。
2. xAgent 查找并复制对应 Skill 到会话目录，作为更新草稿。
3. 你说明要修改的目标、问题或新增要求。
4. xAgent 基于草稿修改 Skill 内容。
5. 你检查草稿，确认是否还需要继续调整。
6. 确认后发布更新。

更新 Skill 同样建议使用推理能力更强的模型。如果模型能力不足，Skill 的逻辑性、边界判断、步骤组织和长期可维护性都会明显下降。

## 一个简单模板

可以按下面结构整理：

```text
名称：
用于：
用户需要提供：
处理步骤：
输出格式：
需要先确认的动作：
不要做的事情：
完成标准：
示例任务：
```

## 测试 Skill

创建或更新后建议用真实但低风险的任务测试：

1. 准备一份典型材料。
2. 在 Agent 会话中明确要求使用这个 Skill。
3. 检查输出是否符合预期。
4. 如果步骤不稳定，回到 Skill 构建或更新会话中补充限制、示例或质量标准。
5. 如果 Skill 太复杂，拆成多个更小的 Skill。

## 发布与维护

Skill 的发布流程建议先从个人开始，再进入公共库：

1. 创建或导入 Skill 后，先形成草稿。
2. 草稿确认可用后，发布到私人库，成为当前用户的个人 Skill。
3. 个人 Skill 先在自己的真实任务里使用和测试。
4. 确认稳定、通用、无敏感信息后，再提交公共库审批。
5. 管理员审核通过后，才进入公共库，供所有用户查看和使用。

公共 Skill 对所有用户可见，xAgent 当前不对公共 Skill 做复杂访问权限限制。公共 Skill 应尽量面向多人，不要写个人习惯、临时路径、私有账号信息、客户专属流程或不希望其他用户看到的内容。有特殊要求的 Skill，请保存在个人 Skill 库下。

提交公共库审批时，系统会复制当前正式个人 Skill 的快照；后续继续修改个人 Skill，不会自动改变已经提交的审批内容。后续如果发现输出质量下降、外部系统变化或业务流程变化，需要及时更新。

Skill 发布后不是一次性结束。你可以在原来的 Skill 构建会话里继续说明问题、补充测试样例和修改要求，让 xAgent 协助优化；也可以新建一个会话，围绕已经发布的 Skill 做下一轮调整、测试和发布。
