# xAgent 文档写作任务

你正在为 xAgent 编写官方文档。

## 基本要求

- 使用中文 Markdown。
- 语气客观、清晰、工程化。
- 不写营销文案。
- 不编造代码中不存在的功能。
- 不把规划能力写成已实现能力。
- 如果功能尚未完全实现，明确标记为“规划中”或“实验性”。
- 所有术语必须与 `docs/reference/glossary.md` 保持一致。
- 修改功能行为说明时，必须同步 `docs/changelog.md`。
- 新增术语时，必须同步 `docs/reference/glossary.md`。
- 写具体行为、配置、API、命令前，必须查代码或已有稳定文档；没有依据时只写概念边界或标记为 `planned` / `experimental`。

## 文档结构

每篇文档尽量包含：

1. 适用对象
2. 这是什么
3. 什么时候使用
4. 基本用法
5. 常见问题
6. 相关文档

## 禁止事项

- 不允许生成未验证的 API。
- 不允许编造配置项。
- 不允许把架构设想写成已实现。
- 不允许随意改动导航结构。
- 不允许修改部署配置，除非任务明确要求。
- 不允许引入复杂依赖，除非任务明确要求。

## 修改范围

默认只允许修改：

- `docs/`
- `blog/`
- `sidebars.ts`
- `docusaurus.config.ts`
- `README.md`
- `CONTRIBUTING.md`
- `.codex/prompts/`

如需修改其他文件，必须说明原因。
