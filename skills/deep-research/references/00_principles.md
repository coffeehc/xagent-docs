# 深度研究原则

## 1. 深度研究不是普通搜索

普通搜索通常回答“是什么”。深度研究要回答：

- 为什么是这样；
- 证据是否可靠；
- 不同来源是否冲突；
- 结论是否能支持决策；
- 哪些不确定性仍然存在；
- 后续应如何验证。

## 2. 研究过程必须可恢复

长时研究会遇到上下文衰减、工具失败、网页变化、文件解析失败、信息冲突等问题。因此必须落盘：

- query log
- source registry
- download registry
- parse registry
- evidence ledger
- gap matrix
- contradiction log
- claim map
- audit report

## 3. 搜索不是证据

搜索结果只用于发现来源。证据必须来自已 fetch、已下载、已解析或已人工检查的来源。

## 4. 来源要先治理

每个来源都要判断：

- 谁发布；
- 什么时候发布；
- 是否一手；
- 是否有利益相关；
- 是否能验证；
- 是否与研究范围一致。

## 5. 证据要绑定结论

最终报告中的每个重要结论都应能追溯到 evidence ID。建议、判断和推断也要说明依据。

## 6. 冲突不是异常

真正的深度研究经常发现来源冲突。冲突应该被记录、解释，并在必要时保留在最终报告中。

## 7. 不确定性要显性化

如果证据不足，应明确说证据不足。不要用模糊措辞掩盖不确定性。
