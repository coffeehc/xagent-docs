# Report Template

## 通用深度研究报告模板

```markdown
# <Title>

## Executive Summary

用 5–10 条 bullet 给出核心结论。每条核心结论应能回溯到 evidence IDs 或引用。

## Scope and Method

- Research question:
- Audience:
- Decision context:
- Geography:
- Time window:
- Depth:
- Sources used:
- Method:
- Limitations:

## Key Findings

| Finding | Evidence | Confidence | Notes |
|---|---|---|---|
| ... | E0001, E0003 | high | ... |

## Evidence Review

按主题整理证据，不要简单堆来源。

## Detailed Analysis

结合证据回答研究问题。

## Contradictions and Uncertainties

说明冲突、口径差异、未解决问题和证据限制。

## Recommendations / Implications

如果用户需要决策建议，给出建议、适用条件、风险和后续验证步骤。

## Source Notes

列出主要来源和质量说明。

## Appendix

可放查询日志、来源清单、评分矩阵、方法细节等。
```

## 表格建议

### 对比矩阵

```markdown
| Dimension | Option A | Option B | Option C | Evidence | Notes |
|---|---|---|---|---|---|
```

### 时间线

```markdown
| Date | Event | Source | Significance |
|---|---|---|---|
```

### 证据矩阵

```markdown
| Claim | Evidence IDs | Source tier | Confidence | Caveat |
|---|---|---|---|---|
```

### 推荐矩阵

```markdown
| Recommendation | Rationale | Evidence | Risk | When to choose |
|---|---|---|---|---|
```

## 写作规则

- 对当前状态使用明确日期。
- 区分事实、估计、推断和建议。
- 不隐藏不确定性。
- 不使用“显然”“毫无疑问”等过度确定措辞，除非证据非常强。
- 引用不要集中放在文末；关键事实附近要能看到来源或 evidence IDs。
