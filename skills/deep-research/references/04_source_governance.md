# Source Governance

## 来源注册原则

任何将被用于证据、引用或结论的来源必须先进入 `source_registry.jsonl`。

## 来源状态

| Status | Meaning |
|---|---|
| candidate | 搜索发现但未阅读 |
| approved | 已阅读，质量足够，可用于证据 |
| rejected | 不适合使用 |
| needs_review | 可能有用，但需要更多验证 |

## 来源等级

| Tier | 类型 | 示例 | 使用方式 |
|---|---|---|---|
| 1 | 一手/权威 | 官方文档、标准、监管文件、论文、财报、原始数据、产品发布说明 | 优先使用，可支撑关键事实 |
| 2 | 高质量二手 | 权威媒体、行业报告、专家文章、学术综述 | 可用于背景、解释和交叉验证 |
| 3 | 有用但有偏 | 厂商博客、供应商白皮书、社区讨论、非独立 benchmark | 可作为补充，需标记偏差 |
| 4 | 弱来源 | 论坛、社媒、SEO 聚合、无署名文章、不可验证页面 | 一般不用于关键结论 |

## 质量评分

建议 1–5 分：

| Score | Criteria |
|---:|---|
| 5 | 一手、权威、当前、具体、方法清晰 |
| 4 | 可信二手来源，信息具体，来源透明 |
| 3 | 相关但有偏差或方法不完整 |
| 2 | 可疑、过旧、缺少作者/日期/方法 |
| 1 | 不可验证、明显 SEO、误导性或低质量 |

## 领域特定优先级

### 技术研究

优先级：

1. 官方 docs / specs / release notes。
2. 标准组织或协议规范。
3. GitHub repo、issues、PR、changelog。
4. 论文、benchmark、incident report。
5. 高质量工程博客。
6. 社区讨论。

### 市场研究

优先级：

1. 监管数据、官方统计、公司财报、投资者材料。
2. 行业研究机构。
3. 权威媒体和专家访谈。
4. 厂商资料。
5. 二手聚合。

### 政策/法规研究

优先级：

1. 法律原文、监管机构、官方 guidance。
2. 判例、执法公告、咨询文件。
3. 律所/合规机构分析。
4. 媒体解释。

## 单一来源风险

重大结论不能只由一个来源支撑，除非该来源是唯一权威原文。若只能找到单一来源，必须在报告中说明。

## 厂商来源处理

厂商来源可用于：

- 产品功能。
- 官方定价。
- 路线图声明。
- 案例说明。

厂商来源不宜单独用于：

- 市场份额判断。
- 与竞争对手的优劣比较。
- 性能或成本优势结论。
- 安全/可靠性优势结论。
