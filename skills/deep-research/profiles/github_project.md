# Profile: GitHub / Open Source Project Research

用于开源项目、GitHub repo、社区健康度、技术风险和采用建议研究。

## 子问题模板

- 项目目标和核心能力是什么？
- repo 活跃度如何？
- release 节奏如何？
- issue/PR 响应情况如何？
- maintainer 和治理结构如何？
- 文档、测试、CI、兼容性如何？
- 安全公告和历史漏洞如何？
- 许可证和商业使用风险是什么？
- 生产采用案例有哪些？

## 来源优先级

1. GitHub repo README、docs、releases、tags。
2. issues、PR、discussions。
3. security advisories。
4. 官方网站和文档。
5. 生产案例和工程博客。
6. 社区讨论。

## 必备证据

- 最近 release 日期。
- stars/forks 可作弱指标，但不能单独判断质量。
- open issues 和 PR 状态。
- commit 活跃度。
- license。
- security policy。

## 特别注意

- stars 不是生产成熟度。
- 单一 maintainer 是治理风险。
- 没有 release 不一定表示不活跃，但必须解释。
- issue 数量高可能表示流行，也可能表示质量问题。
