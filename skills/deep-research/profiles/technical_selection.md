# Profile: Technical Selection

用于技术选型、架构决策、框架比较、数据库/中间件/云服务评估。

## 子问题模板

- 要解决的技术问题是什么？
- 候选方案有哪些？
- 每个方案的核心架构和适用场景是什么？
- 性能、可靠性、可扩展性、安全性如何？
- 运维复杂度、成本和团队学习曲线如何？
- 生态、社区、商业支持、版本活跃度如何？
- 已知限制、事故、迁移风险是什么？
- 推荐方案和适用条件是什么？

## 来源优先级

1. 官方 docs、specs、release notes。
2. GitHub repo、issues、PR、changelog。
3. 标准组织和协议规范。
4. 论文、benchmark、incident report。
5. 高质量工程博客和案例研究。
6. 社区讨论。

## 必备表格

```markdown
| Dimension | Option A | Option B | Option C | Evidence | Notes |
|---|---|---|---|---|---|
```

## 特别注意

- benchmark 只能说明测试条件下的结果。
- 不同版本能力差异必须标注。
- 技术推荐必须绑定约束条件。
- 不要把 popularity 当成 suitability。
