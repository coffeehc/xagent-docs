# Performance Review

## Purpose

Create fair, specific, evidence-backed performance material that separates impact, behavior, examples, growth areas, and next expectations.

## Use When

Use for:

- self-review drafts
- manager review drafts
- promotion narratives
- 360 feedback
- calibration notes
- development plans
- performance improvement drafts

## Material Gate

Identify:

- review period, role expectations, goals, examples, metrics, feedback, and audience
- whether the output is self-review, manager review, promotion, coaching, or HR-sensitive material
- evidence and missing evidence

Do not invent performance examples, ratings, compensation outcomes, or HR decisions.

## Workflow

1. Organize evidence by goals, responsibilities, behaviors, and impact.
2. Separate outcomes, actions, collaboration, and growth areas.
3. Write feedback with specific examples and observed behavior.
4. Balance strengths, gaps, and future expectations.
5. Avoid vague labels, personality judgments, and unsupported claims.
6. Use `html-report-builder` only for aggregate team review summaries or calibration reports when appropriate.

## Checks

- Claims have examples or metrics.
- Feedback describes behavior and impact.
- Growth areas include actionable expectations.
- Tone fits the audience and sensitivity.
- HR/legal-sensitive conclusions are not overclaimed.

## Outputs

- self-review
- manager review
- promotion narrative
- 360 feedback summary
- development plan
- calibration note
- performance improvement draft

## Boundary

This Skill owns performance writing structure, evidence framing, and feedback clarity. It does not own HR approval, ratings, compensation, employment decisions, legal advice, or personnel-system updates.

It must not run shell, Python, Node, npm, curl, MCP, HRIS, ATS, payroll, email, calendar, package install, or external API commands by itself; those actions belong to XAgent tools visible in the current session.
